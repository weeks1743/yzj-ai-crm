# 0.7.1 元数据驱动的记录查询条件识别

## 版本目标

- 修复“查询安徽省的客户”被错误映射为标题/名称模糊搜索的问题。
- 将修复扩大为通用机制：凡是影子元数据声明为可查询的字段，都优先尝试提取为结构化查询条件。
- 保持 Agent core 业务无关，不新增 `scene.*` 运行时技能，不改 `apps/prototype`。

## 关键变更

- 在 CRM Agent Pack 内新增记录搜索条件提取流程，基于对象字段元数据、字段标签、语义槽、选项和控件类型构建字段目录。
- record search 输入构造顺序调整为：上下文绑定条件、元数据字段条件、名称/标题兜底。
- 支持显式字段条件，例如“省是安徽”“客户状态为潜在客户”“行业为制造业”“联系人手机为 13800138000”。
- 支持保守隐式条件，例如“安徽省的客户”“启用客户”；普通文本字段不做隐式字段识别。
- 对歧义字段和值未命中选项的情况返回澄清，不降级为 `_S_NAME` 或标题搜索。
- 在 `agentControl.searchExtraction` 中保留字段提取摘要，便于用户运行洞察和后台排查。
- 记录读取完成消息中的结果 JSON 改为完整输出，不再截断用户可见的查询结果。
- 记录查询与详情结果新增 A2UI v0.9 `uiSurfaces` 输出，用户端使用 `@ant-design/x-card` 渲染为空态、列表或详情卡片，原始 JSON 改为排障信息。
- 修复“查询所有客户/全部客户/客户列表”以及“查询安徽省所有客户”被识别成名称包含范围词的问题，范围词不再进入 `_S_NAME` 或标题兜底。
- 多条记录查询结果从卡片堆叠改为 Ant Design 表格列表，便于扫描客户、状态、地区、联系人、负责人和操作。
- 修复同一会话内工具进入澄清/写入等待态时沿用上一轮 A2UI `uiSurfaces` 的问题，避免旧客户卡片污染新回复。
- 修复用户 AI 端会话切换污染问题：会话运行区按 `activeConversationKey` 强制重新挂载，消息 store、运行洞察、请求 provider 与当前会话绑定。
- assistant-web 本地聊天存储升级到 `v3`，启动时清理 `v1/v2` 历史数据，清除已污染的本地会话记录，仅保留固定会话入口。
- 消息持久化增加当前会话快照校验，旧会话切换过程中的消息快照不会写入新会话。
- A2UI 记录列表改为真实动态分页：首屏和翻页均按当前 filters 调用后端分页查询，表格每页展示 5 条，翻页在原表格内刷新数据，不新增聊天消息。
- 修复记录详情点击暴露内部 `formInstId` 的问题：用户可见消息显示业务标题，内部 ID 通过隐藏 `clientAction` 传递给后端。
- 写入参数抽取升级为元数据驱动，支持商机预算、预计成交时间、销售阶段等明确字段标签写入表达，避免“更新预算为5000000”生成空 `params`。
- 运行洞察新增“运行流程”诊断页签，用流程图展示意图、上下文、工具输入、工具结果、策略守卫和最终状态，便于定位空写入守卫等问题。
- 修复商机列表结果在后续对话中被异步请求/组件状态污染的问题：A2UI surface 按 `surfaceId` 强制隔离挂载，用户端同一会话内禁止多请求并发。
- 修复“更新销售负责人为陈伟堂”被 LLM 误判为 company 后误走公司研究的问题：写入类请求优先承接当前记录上下文，外部公司研究只响应分析/研究类意图。
- 写回确认改为查找最近待确认 trace，不再只依赖最后一条 Agent trace，避免慢请求后返回冲掉确认上下文后点击“确认”无响应。
- 上下文读取会跳过形如内部记录 ID 的伪公司主体，避免历史误路由的公司研究结果污染后续记录更新承接。
- 人员选择控件写入改为先查组织员工目录：所有可写 `personSelectWidget` 字段都会基于当前 `eid + app_id` 查询 `org_employees`，唯一命中写入 `open_id`，不再把姓名当普通文本传给轻云。
- 人员字段支持重名澄清：姓名包含匹配命中多人时返回单选问题卡，候选展示“姓名 + 手机 + 邮箱”，用户选择后继续生成写回确认；未命中时要求补充姓名、手机号或邮箱，不调用写入预览。
- 写回确认正文不再输出调试 JSON，业务用户只看到可读字段和值；完整 `open_id`、工具输入和轻云返回仍保留在运行洞察/原始追踪中排查。
- 补字段卡中的人员选择与关联记录字段升级为远程检索下拉选择：`personSelectWidget` 查组织员工，`basicDataWidget` 按关联表单映射到 CRM 影子对象后查记录，禁止把自由文本直接提交为引用字段值。
- 新增只读候选接口 `POST /api/agent/meta-question-options`，用于问题卡按字段查询员工或记录候选；关联客户搜索采用综合匹配，输入“安徽”会同时尝试名称/标题和元数据字段条件。
- 修复组织员工候选因应用 ID 变更返回空的问题：员工目录检索先查当前 app，同 app 未同步数据时兜底查同一 `eid` 下历史 app 的组织员工数据，不跨租户。
- 修复补字段卡提交文本污染查询条件的问题：用户气泡只展示候选名称，不再把客户状态、客户类型等候选详情拼进自然语言；结构化提交即使遇到旧卡或运行检查点丢失，也不会降级为客户标题/字段查询。
- 补字段卡增加只读保留态：已提交、已处理或已过期的卡片继续留在历史消息中用于回看，只有当前最新待补充卡可编辑提交，避免录入卡片被后续回复替换或误操作。

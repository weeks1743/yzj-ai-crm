# 0.7.1 元数据驱动的记录查询条件识别

## 版本目标

- 修复“查询安徽省的客户”被错误映射为标题/名称模糊搜索的问题。
- 将修复扩大为通用机制：凡是影子元数据声明为可查询的字段，都优先尝试提取为结构化查询条件。
- 保持 Agent core 业务无关，不新增 `scene.*` 运行时技能，不改 `apps/prototype`。

## 关键变更

- 在 CRM Agent Pack 内新增记录搜索条件提取流程，基于对象字段元数据、字段标签、语义槽、选项和控件类型构建字段目录。
- record search 输入构造顺序调整为：上下文绑定条件、元数据字段条件、名称/标题兜底。
- 支持显式字段条件，例如“省是安徽”“客户状态为潜在客户”“行业为制造业”“联系人手机为 13800138000”。
- 支持保守隐式条件，例如“安徽省的客户”“启用客户”；普通文本字段不做隐式字段识别。
- 对歧义字段和值未命中选项的情况返回澄清，不降级为 `_S_NAME` 或标题搜索。
- 在 `agentControl.searchExtraction` 中保留字段提取摘要，便于用户运行洞察和后台排查。
- 记录读取完成消息中的结果 JSON 改为完整输出，不再截断用户可见的查询结果。
- 记录查询与详情结果新增 A2UI v0.9 `uiSurfaces` 输出，用户端使用 `@ant-design/x-card` 渲染为空态、列表或详情卡片，原始 JSON 改为排障信息。

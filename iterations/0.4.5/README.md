# 0.4.5 Agent 治理重构

## 版本目标

- 移除“场景技能”作为后台一等治理入口。
- 将管理员后台调整为业务无关的 `Agent 治理`。
- 用 `IntentFrame -> TaskPlan -> Tool Selection -> ExecutionState` 作为运行口径。

## 范围

- 管理员后台：
  - 删除菜单“技能与编排 / 场景技能”。
  - 恢复 `技能目录`，保留 `记录系统技能` 与 `外部技能` 两个日常目录入口。
  - 新增 `Agent 治理` 一级菜单。
  - 固定五个子页：`工具与对象`、`会话任务`、`计划模板`、`策略与确认`、`运行观测`。
  - 删除 `AI 资产` 一级菜单，不再展示 `录音分析资产`、`公司研究快照`、`拜访材料结果` 三个菜单。
  - 系统设置只保留基础接入配置和企业 PPT 模板。
- 共享数据：
  - 新增 `agentToolRows`、`agentPlanTemplates`、`agentPolicies`、`agentRuntimeTraces`。
  - 保留旧 Playbook 数据作为迁移来源，但前端不再以“场景技能”展示。
- 文档：
  - 更新对话层与编排口径。
  - 将场景技能口径调整为计划模板与工具编排原则。
  - 新增通用 Agent 治理模型文档。

## 验收项

- 后台菜单不再出现“场景技能”。
- 后台菜单保留“记录系统技能”和“外部技能”，便于管理员查看现有技能目录。
- 后台出现“Agent 治理”及五个子页，其中“会话任务”并入 Agent 治理。
- 后台不再出现“AI 资产”一级菜单，也不再出现“录音分析资产 / 公司研究快照 / 拜访材料结果”菜单。
- “工具与对象”可看到记录工具、外部工具和 Meta 工具。
- “计划模板”只展示推荐计划，不表达为固定工作流或可调用场景技能。
- “策略与确认”承接写回确认、安全守卫和证据要求。
- “运行观测”展示 IntentFrame、TaskPlan、ExecutionState 与工具调用链。
- 系统设置不再重复展示模型/provider、观测、安全与运营治理。
- 访问 `/agent-governance/*` 不再因为旧 `/skills/*` 兼容路由嵌入菜单树而触发运行态异常。

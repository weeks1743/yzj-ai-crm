# 0.6.3 实测上下文稳态与客户旅程摘要

## 版本目标

- 修复真实多轮对话里上下文主体从明确客户退化成“这个”的问题。
- 修复“这个客户的联系人 / 商机 / 跟进记录”没有利用上下文主体做关系型搜索的问题。
- 补一条业务包层的上下文摘要能力，让金蝶苏州销售员可以围绕当前客户查看旅程概览和下一步推进建议。

## 实施范围

- `agent-runtime` / `agent-service` / `agent-run-repository`：
  把稳定上下文主体作为运行资产持久化，不再只依赖历史 `legacy intent` 反推。
- `crm-agent-pack`：
  - record read 成功后写回明确 `ContextFrame`
  - 关系型对象搜索通过工具元数据绑定当前客户主体
  - 新增 `meta.context_summary`，汇总客户详情、联系人、商机、跟进记录并输出销售视角摘要
- 单测补充上下文持久化、关系型搜索、多轮客户旅程摘要。

## 依赖框架

- LangGraph
- LangChain JS
- 现有 Shadow Metadata / LightCloud 真实读链路

## 验收结果

- `pnpm --filter @yzj-ai-crm/admin-api test`：通过，包含原 40 个 Harness 场景与新增上下文主体 / 关系搜索 / 客户旅程摘要单测。
- `pnpm --filter @yzj-ai-crm/admin-api build`：通过。
- 真实联调已验证：
  - `查询客户 上海松井机械有限公司` 命中现有客户
  - `查询这个客户的联系人` 命中 4 条
  - `查询这个客户的商机` 命中 4 条
  - `查询这个客户的跟进记录` 命中 2 条
  - `请从金蝶苏州销售员视角，列出这个客户的客户旅程` 返回客户旅程摘要
  - `我下一步应该怎么推进` 返回推进建议摘要

## 未完成项与下一步计划

- 若需要把“客户旅程摘要”进一步扩展成跨更多业务包的通用摘要工具，再单独沉淀更抽象的摘要元数据。
- 本轮不改记录系统 Skill 语义，只在主 Agent 运行层和 CRM 业务包补齐上下文与编排。

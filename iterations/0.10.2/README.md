# 0.10.2 本地产品问题修复

## 版本目标

- 修复记录录入与展示中的内部 ID 外泄问题，尤其是商机、商机跟进记录、所属部门等用户不可读字段。
- 精简用户 AI 端公司研究证据卡，恢复图片点击预览能力。
- 用户侧屏蔽 PPT 生成概念；确认企业模板链路不稳定后，后端保留诊断能力但不再在普通用户界面暴露。
- 优化录音资料包、证据卡布局、会话侧栏与回答纯净度，避免用户看到技术状态、低价值描述或本地调试身份文案。

## 范围

- 用户 AI 端：公司研究资料卡、图片预览/下载、录音资料卡跳转、录音查看页 loading 过渡。
- 管理端/后端共享记录结果模型：记录标题、关联对象、部门和人员等展示值清洗。
- skill-runtime：Docmee super-ppt 失败信息归一化与无模板 smoke 验证，当前不暴露到普通用户侧。

## 关键修复

- 记录结果与写入预览统一过滤纯内部 ID、`ID的商机`、`ID的商机跟进记录` 等不可读标题；新增商机/跟进记录标题优先取商机名称、跟进内容等业务可读字段，旧数据展示层也做二次兜底。
- 轻云 live `widgetValue` 里的部门字段若只有 ID，展示为 `已选择部门`；人员字段继续解析组织员工，查不到时展示 `已绑定人员`。
- 公司研究证据卡只保留业务标题、派生动作和图片结果区，不展示版本、引用合并、命中片段、anchor/chunk/artifact 等技术信息。
- 图片生成成功后，资料卡内图片使用 Ant Design `Image` 预览，图片右上角提供下载按钮；“查看完整研究”是唯一打开 Markdown Drawer 的入口，图片预览/关闭不再触发 Markdown。
- 用户 AI 端移除证据卡与 Markdown Drawer 内的 PPT 生成 / 下载 / 失败展示入口，避免把不稳定能力暴露为用户侧概念。
- 只有 `recording_material` 录音资料包显示“打开录音分析”并跳转录音分析查看页；`analysis_material` 等其他资料仍显示“查看完整研究”，且不提供生成图片或 PPT 动作。
- 资料卡改为横向网格布局，普通资料尽量一排展示，带图片的资料卡单独占一行；卡片主体不再负责打开 Markdown，只保留明确操作按钮。
- 普通 Markdown 资料卡恢复点击整卡直接侧滑打开；图片区域与下载按钮继续阻止冒泡，避免图片预览/关闭误触发 Markdown。
- 用户输入框暂时屏蔽语音输入；右上角移除固定“公司研究入口”标签。
- 左侧会话列表接入 Ant Design X `Conversations` 菜单，用户创建的会话支持重命名；侧栏品牌向左上收紧，左下角仅保留头像、姓名和设置入口，不显示角色/SOUL 调试文案。
- 拜访需求类回答只保留“客户主要需求”正文，不再输出“拜访需求摘要”“证据来源”“记录系统补充”“公司研究背景”“分析资料状态”等负担信息。
- 当拜访需求资料片段不足以直接抽取需求时，会继续从完整分析 Markdown、系统内跟进记录等业务文本中提炼需求，避免只返回空兜底。
- `trace-agent-83d157b0` 的根因定位为跨系统 appId 隔离：AI 销售系统使用 `YZJ_APP_ID=501037729`，记录系统资料归档使用 `YZJ_LIGHTCLOUD_APP_ID=501037649`，原检索按 `appId` 强过滤导致已归档录音/分析资料无法命中。
- 资料检索新增同租户精确业务锚点兜底：仅当同一 `eid` 且命中 `customer/opportunity/followup/company` 精确锚点时，允许从记录系统 appId 下读取只读资料资产；不做硬编码 appId，也不放开跨租户或无锚点检索。
- 拜访需求路由改为 CRM 业务包内的语义维度分析，综合问句、IntentFrame 目标、拜访/沟通场景、客户诉求/关注点、下一步建议和经营判断信号判断 `visit_needs / next_step / journey`，避免继续堆固定问句特判。
- 需求抽取增加客户诉求语义边界，只接收客户侧提出、希望、关注或具体业务能力需求；过滤“建议下一步”“补充省市区”“打开完整资料”等销售维护建议，避免把内部动作误当客户需求。
- 客户信息更新新增局部“更新前定位”守卫：只有已进入 `record.*.preview_update` 且没有记录上下文时，才按用户输入的客户名先查记录；唯一命中后继续更新预览，多条命中让用户选择，未命中时提示换更完整名称，不再要求普通用户输入内部 ID。
- 当用户先说“更新客户”再补客户名但没有说明修改字段时，只绑定目标客户并展示更新字段选择卡；卡片带目标可读名称、可提交字段问题和业务字段标签，不再调用空写入预览，也不向用户展示工具名、`readyToSend` 或内部诊断。
- 更新字段选择卡复用通用 Meta Question Card / `provide_input` 机制，覆盖客户、联系人、商机、拜访记录；支持状态/阶段枚举、文本字段、手机号、基础资料引用、负责人选择和多字段一次提交。
- 更新字段选择卡改为卡内分步抽屉式体验：后端返回全部可提示写入字段并补齐当前值，前端默认展示常用字段、支持字段搜索和更多字段展开，用户选择字段后再填写新值；提交时只发送已选择、非空且相对当前值已变更的字段。
- 更新字段选择卡的问题来源改为工具白名单与轻云元数据可写字段并集，补上“备注”等未进入固定 `writableParams` 但可写、可提示的字段；字段搜索按业务标签命中。
- “已选择修改”区的字段行本身可点击或键盘激活，用户可从下方已选列表直接切换当前编辑字段，不必回到上方字段列表重复寻找。
- 省、市、区等 `publicOptBoxWidget` 公共选项字段不再渲染为枚举按钮，改用 `field_option` 远程搜索候选，避免大量公共选项一次性铺满卡片。
- 更新字段卡输入区不再把已有值预填为新值；文本、日期和远程搜索控件默认留空，已有值只在字段行/已选行展示，避免重复输入和下拉遮挡，枚举字段保留“当前”标记。
- 更多字段排序改为必填优先、已有值优先，再按字段语义排序；“展开更多字段”每次增量显示 5 个，避免一次性列出全部字段。
- 日期字段修改改用 Ant Design `DatePicker`，不再使用普通文本输入框。
- 当前值优先来自轻云详情读取，失败时回退搜索命中摘要；枚举字段按真实 option key/value 预选，人员和基础资料字段只展示可读名或“已绑定人员 / 已关联记录”，不暴露 `openId`、`formInstId`。
- 取消补充/预览流程后只显示“已取消本次录入”的普通回复，trace 保留 `cancelled`，但后端不再返回可渲染字段卡，前端也不会把旧卡片显示成“已处理”。
- 更新预览前会剥离“等于目标名称的身份字段”回显，例如“更新联系人李玲玲”只用于定位联系人，不会被误当成“把联系人姓名更新为李玲玲”；电话字段语义也收紧，`手机号` 不再同时污染 `办公电话`。
- 当前输入包含明确客户名时，不复用上一轮“客户/联系人”等泛化上下文作为记录 ID，避免把补充客户名误判成已有上下文更新。
- 更新前定位不改全局工具路由：录入客户仍走 `preview_create`，查询客户仍走 `record.customer.search`，综合资料问题仍走 `meta.context_summary` / `artifact.search` 并继续消费公司研究、录音资料包和分析结果。
- Artifact 详情响应补充版本 metadata，前端可从录音资料 metadata 中读取 `recordingTaskId` 并跳转 `/api/recording-audio-tasks/:taskId/meeting-viewer`。
- 录音查看页打开前先进入 `/recording-viewer-loading`，展示“录音分析正在打开，请稍候”，再跳转服务端 meeting-viewer 地址。
- 本地复测确认当前企业 PPT 模板会让 Docmee 官方布局长期停留在部分页，生成链路先不注入企业模板 ID；后台模板记录保留，后续再做模板健康检查与恢复开关。
- Docmee 官方布局 SSE 中断后继续轮询官方任务结果；当无模板链路完成布局时继续生成 PPT，仍不做旧式默认模板降级。

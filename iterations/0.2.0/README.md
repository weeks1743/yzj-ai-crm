# 0.2.0 版本说明

## 版本目标

在 `0.1.1` 双系统正式原型基础上，补齐一条可真实联调的“系统基础设置 + 云之家组织同步”后端链路：

- 新增独立 `admin-api` 服务承载系统基础设置读取与组织同步
- 使用本地 `.env` 管理真实云之家凭据，不把明文密钥提交到仓库
- 使用独立 SQLite 数据库存储同步运行记录与在职人员快照
- 管理员后台系统设置页接入真实 API，并在组织同步页提供手动同步按钮

## 为什么本次是 0.2.0

本轮不只是小修补，而是新增一层真实后端工作区、持久化与测试体系，并把管理端设置页从纯 mock 推进到可真实联调的产品原型，因此按中型迭代升级为 `0.2.0`。

## 本次范围

- 新增 `apps/admin-api`
- 新增根级 `.env.example`
- 新增组织同步 SQLite 持久化
- 实现云之家 `resGroupSecret` token 获取
- 实现“查询全部在职人员信息”分页同步
- 管理员后台接入：
  - `租户 / 应用`
  - `云之家接入`
  - `组织同步`
- 新增 admin-api 自动化测试
- 更新根 README 与正式设计文档中的当前阶段口径

## 关键能力

### 1. admin-api

- 使用 TypeScript + Node 实现
- 读取本地 `.env`
- 提供设置与组织同步 HTTP 接口
- 使用 SQLite 保存：
  - `org_sync_runs`
  - `org_employees`

### 2. 组织同步

- 只做手动触发
- 只做一次性全量同步
- 只同步在职人员
- 不做 cron
- 不做增量
- 不做部门、角色、上下级关系同步

### 3. 管理端设置页

- `租户 / 应用` 改为真实后端读取
- `云之家接入` 改为真实后端读取，并展示脱敏后的凭据状态
- `组织同步` 改为真实操作页，支持：
  - 查看最近同步状态
  - 查看在职人员总数
  - 查看最近运行记录
  - 触发手动同步

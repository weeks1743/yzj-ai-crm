# 0.8.14 客户列表摘要字段恢复

## 目标

- 修复 `查询客户` 列表中客户名、状态、地区、负责人等字段为空或显示为“客户记录”的问题。
- 保持 0.8.13 的内部 ID 隐藏策略，用户可见层继续不展示 `formInstId`。

## 范围

- 记录结果展示字段读取不再只依赖完整 `fieldContent`。
- 当轻云列表查询只返回 `important` 摘要字段时，后端从 `important` / `rawRecord.important` 补齐：
  客户名称、客户状态、客户类型、地区、负责人等展示字段。
- 列表、卡片、上下文摘要统一复用同一套展示字段读取逻辑。

## 验收

- `查询客户` 返回多条记录时，列表标题展示客户业务名称，不展示“客户记录”或内部 ID。
- 状态列可展示客户状态/客户类型标签。
- 地区列可展示省、市、区。
- 负责人列可展示负责人或销售负责人。
- 后续点击“查看详情”仍通过隐藏 `formInstId` 读取详情。

# 0.7.2 新增记录名称占位词修复

## 版本目标

- 修复“新增客户”被识别为客户名称的问题。
- 保持记录创建的名称抽取为通用机制，不在主 Agent 内核硬编码 CRM 场景分支。
- 保持用户 AI 端与管理员后台框架基线不变，本轮只修复 Agent 记录写入参数抽取。

## 范围说明

- 记录创建意图的目标名称清洗。
- 创建预览前的主题名称参数和查重条件生成。
- 后端 Agent 运行时回归测试。

## 关键能力

- 仅当用户提供真实名称时，才填充 `customer_name`、`contact_name`、`opportunity_name` 或 `followup_record`。
- “新增客户”“创建联系人”“新增跟进记录”等动作/对象占位表达不会进入名称字段。
- “新增客户 测试客户 联系人姓名：陈丽”仍保留“测试客户”作为客户名称，并继续抽取后续字段。

## 依赖框架

- 主 Agent 仍通过业务无关 `IntentFrame`、`ContextFrame`、`ToolDefinition` 与 CRM Agent Pack 协作。
- 未新增 `scene.*` 运行时技能。

## 验收结果

- 已通过：`新增客户` 不再生成 `customer_name: 新增客户`，也不会触发按“新增客户”查重。
- 已通过：`新增客户 测试客户` 仍可生成客户名称并进入写回确认/补字段流程。
- 已通过：`新增跟进记录 已电话沟通` 保留真实跟进内容，`新增跟进记录` 这类对象名不再充当正文。
- 已通过：`pnpm --filter @yzj-ai-crm/admin-api test`。
- 已通过：`pnpm --filter @yzj-ai-crm/admin-api build`。

## 未完成项

- 无。

# 0.8.4 公司研究在 AI 销售工作台完整跑通

## 版本目标

- AI 销售工作台只开放“公司研究”一个可用入口。
- 使用 `/公司研究 上海松井机械有限公司` 跑通公司研究测试链路。
- 已有有效公司研究资料时默认复用，不重复调用外部研究服务。

## 范围

- 收敛 `assistant-web` 工作台首页、斜杠菜单和左侧固定会话。
- 在 `admin-api` 公司研究工具执行前增加已有资料查询。
- 保留旧场景底层类型，不在本轮扩展录音类、问题陈述、价值定位、方案匹配和任务入口。

## 关键页面 / 能力

- AI 销售工作台：
  - 斜杠菜单只显示 `/公司研究`。
  - 首页快捷提示固定包含 `/公司研究 上海松井机械有限公司`。
  - 左侧只保留工作台首页和真实用户会话，不再展示“我的任务”等固定示例入口。
- Agent 运行时：
  - 公司研究先按租户、应用和公司名称查询最新有效公司研究资料。
  - 命中已有有效资料时直接复用，并在工作台展示一张“公司研究资料”卡。
  - 未命中时才调用公司研究服务，成功后写入同一家公司资料的新版本。
  - 失败、运行中或无有效资料的结果不创建降级资料。
  - `/公司研究` 入口只激活技能标签，等待用户手动输入公司全称；`输入公司全称`、`公司全称`、`XX公司` 等占位词不会触发查询、外部服务或资料保存。
  - 公司研究服务返回后，落库前会检查返回内容是否明确指向目标公司，并包含公司概览、业务定位、风险、销售切入点或来源等有效研究信息。
  - 如果真实返回表达未检索到有效公开资料、无法确认公司主体或要求补充公司名，则只保留运行记录和工具调用，不生成资料卡。
  - 普通用户回复不展示工具码、向量状态、内部产物名和英文状态。
  - 资料卡提供“查看完整研究”“生成 PPT”“生成图片”，图片生成成功后展示在当前回答下方。
- 会话交互修正：
  - 空白“新会话”只保留一个；再次点击新会话时复用已有空白会话，不再连续创建多条空白历史。
  - 空白新会话不提前写入远端会话历史，只有首次真实提交后才保存为真实会话。
  - 点击 `/公司研究` 只激活当前输入器技能标签，不再切回左侧固定的“AI 销售工作台”会话。

- 已通过：`pnpm --filter @yzj-ai-crm/admin-api build`
- 已通过：`pnpm --filter @yzj-ai-crm/skill-runtime build`
- 已通过：`pnpm --filter @yzj-ai-crm/assistant-web build`

# 0.2.19 版本说明

## 版本目标

- 将管理后台中的 `记录系统` 独立导航正式收口到 `技能与编排 > 记录系统对象`
- 移除旧对象页承担的数据浏览职责，只保留对象治理工作台
- 保留旧 `/records/*` 地址兼容跳转，避免既有内部链接立即失效

## 范围

- `apps/admin-pro` 后台导航、对象入口、对象详情页职责与相关文案收口
- `packages/shared` 与 `apps/assistant-web` 中涉及“记录系统页 / 对象页 / 记录系统技能”的共享提示修订
- `docs/README.md`、`docs/项目公约.md` 同步后台信息架构新口径

## 验收结果

- 左侧导航已移除 `记录系统` 一级分组
- `技能与编排` 下正式入口统一为 `记录系统对象`
- `/records/customers`、`/records/contacts`、`/records/opportunities`、`/records/followups`
  已改为隐藏兼容路由，并自动跳转到对应对象治理页
- 对象详情页继续承接唯一对象工作台职责
  - 保留对象状态、快照版本、字段快照、字典绑定、引用资源与 `shadow.*` 对象能力
  - 不再展示旧的静态业务列表、对象记录抽屉与重复元数据卡
- Dashboard / Workplace / AI 端共享提示中的旧入口文案已统一改为 `对象治理页` 或 `shadow.* 对象能力`
- 正式设计文档已同步“记录系统仍是共享底座，但后台 IA 已并入对象治理”的结论

## 未完成项

- 本轮未新增后台对象数据浏览页，仍以对象治理与能力治理为主
- 本轮未调整后端 `shadow` 接口与技能生成逻辑，仅做前端 IA 与文案收口
- 旧 `/records/*` 兼容跳转仍需保留至少一个版本，后续再评估是否下线

## 下一步计划

- 继续围绕 `记录系统对象` 补充更多对象的真实联调状态与治理信息
- 为场景组装页增加从对象治理页直达依赖矩阵与缺口定位的联动入口
- 视后续产品节奏，再决定是否补回轻量级对象数据浏览能力

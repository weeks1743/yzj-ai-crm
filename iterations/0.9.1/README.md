# 0.9.1 跨会话拜访需求检索修复

## 版本目标

- 修复跨会话问题中公司研究、录音资料包和下游分析资料的组合检索。
- 修复录音先跑下游分析、后新增拜访记录时分析资料未正式归档的问题。
- 优化“上次拜访客户主要提了什么需求”等需求类问题的回答聚焦度。
- 修复录音任务未完成时先写入拜访记录导致正文长期保留“未关联商机”的上下文污染。
- 修复点击“新会话”后被 `/chat` 路由同步或远端会话刷新重新选回固定“AI 销售工作台”的问题。
- 修复新会话内先问拜访需求、再查询唯一客户、随后继续追问时仍无法承接客户主体的问题。

## 范围

- `apps/admin-api`：修复 `meta.context_summary` 的外部资料锚点、需求类摘要输出和录音分析归档。
- `apps/assistant-web`：Artifact Drawer 展示正式锚点，并对正文早于正式关联生成的资料给出重跑提示。
- `docs/`：同步 0.9.1 的数据治理与回答口径。
- 数据修复：用一次性脚本补归档当前已完成但未入库的录音下游分析资料，执行后删除脚本。

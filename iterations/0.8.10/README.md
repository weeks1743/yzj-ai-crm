# 0.8.10 外部信息与系统内记录的会话消费

## 目标

- 修复公司研究后追问“介绍业务 / 有什么业务 / 优势是什么”进入澄清卡的问题。
- 建立业务无关的数据源消费口径：`外部信息` 与 `系统内记录`。
- 将当前公司研究资料归入 `外部信息`，为后续官网、新闻、行业报告、企业查询等系统外来源预留统一入口。

## 范围

- 新增会话侧 `dataSourceScope`：`auto | external_info | internal_records | combined`。
- `外部信息` 当前映射到已有 Artifact 检索能力，主要来源是公司研究资料。
- `系统内记录` 当前映射到记录系统只读工具与上下文摘要能力。
- 融合回答固定输出 `系统内记录`、`外部信息`、`判断建议` 三段。
- 保持 Agent core 业务无关，不新增 `scene.*` 运行时技能。

## 关键场景

- `/公司研究 江苏友升汽车科技有限公司` 后追问 `介绍江苏友升的业务`，路由到外部信息检索，不进入澄清卡。
- `只看外部信息，江苏友升有什么业务？` 只检索外部信息，不触发记录系统。
- `这个客户的商机进展` 默认消费系统内记录，不用外部信息替代。
- `只看系统内记录，江苏友升客户状态是什么？` 只查记录系统，不触发外部信息检索。
- `结合外部信息和系统记录，给我拜访前摘要` 同时消费两类数据并分区输出。

## 验收

- 后端场景测试覆盖外部信息、系统内记录、指定单源和融合消费。
- 回答文案使用 `外部信息` / `系统内记录` 来源标签，不再把公司研究当成唯一的单源消费口径。
- 缺失某一类数据时明确说明缺失来源，不用另一类数据伪装替代。

## 未完成项

- 本轮不新增官网、新闻、行业报告等外部采集源。
- 本轮不改写记录系统 Skill 语义，只通过现有工具契约消费记录数据。

# 0.8.3 公司研究复用与轻量配置静态设计

## 版本目标

- 在现有外部技能详情中补充“公司研究供给”的轻量使用配置设计。
- 明确公司研究有效结果默认复用，不在聊天中重复触发研究。
- 明确无效研究结果只保留任务记录，不进入聊天可引用资料，也不需要管理员标记。

## 范围

- 修改管理员后台 `外部技能 > 公司研究供给` 详情抽屉。
- 将静态“使用配置”从普通信息中拆出，放入独立 Tab，展示聊天使用、重复研究、保存方式、客户关联和无效结果处理口径。
- 将 Skill 类外部能力的“统一 Job 调试台”也拆成独立 Tab，和普通信息分开展示。
- 更新对话层设计文档与项目公约，记录普通管理员配置不暴露内部技术概念。

## 关键页面 / 能力

- `apps/admin-pro` 外部技能详情：
  - 公司研究详情使用 `普通信息 / 使用配置 / 统一 Job 调试台` 三个 Tab，避免普通技能信息、管理员使用规则和调试能力混在一起。
  - 其他可调试的 Skill 类外部能力使用 `普通信息 / 统一 Job 调试台` 两个 Tab。
  - “聊天里可以使用公司研究”默认开启。
  - “已有公司研究时”默认直接使用已有有效结果。
  - “研究结果怎么保存”默认同一家公司保存为一份资料。
  - “怎么和客户资料关联”默认唯一客户自动关联，多个客户让用户选择。
  - “没查到有效资料时”默认只保留任务记录，不作为可用资料。
- “聊天如何组合使用”表格说明记录系统资料与公司研究资料如何分层进入回答。

## 验收结果

- 已通过：`pnpm --filter @yzj-ai-crm/admin-pro build`

## 未完成项

- 本轮只做静态产品设计，不接真实后台规则保存。
- 本轮不实现公司研究复用、无效研究判定或客户自动关联的运行时逻辑。
- 本轮不设计过期提醒，也不设计管理员手动标记无效数据。

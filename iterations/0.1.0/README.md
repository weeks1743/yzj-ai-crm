# 0.1.0 版本说明

## 版本目标

将 `yzj-ai-crm` 从 `0.0.x` 的单应用演示原型，升级为基于官方 `ant-design-pro` + `ant-design/x` 的双系统正式原型：

- 管理员后台
- 用户 AI 端

## 为什么升级到 0.1.0

这轮不是小修小补，而是一次中型重构：

- 工程从单个 `prototype` 应用改为双应用结构
- 文档和项目公约从旧“单产品”口径切换到“双系统”口径
- 页面形态从“手工拼装组件”切换到官方工程范式

因此版本号按项目公约升级为 `0.1.0`。

## 当前设计结论

### 1. 为什么是双系统

因为这轮目标已经不再是“一个 AI 主入口页面演示”，而是同时面向：

- 管理员的治理、配置、可观测性、对象管理
- 用户的对话、任务、场景工作流

这两类人群的页面密度、信息架构和交互方式不同，继续强塞进一个入口只会让产品形态更像演示页。

### 2. 为什么管理员后台必须是 Pro 范式

管理员后台承担的是正式后台职责：

- 记录系统对象治理
- 技能与编排治理
- AI 资产回看
- 可观测性和安全运营

因此页面必须像真实后台，而不是“给 AI 页面加几个表格”。

### 3. 为什么用户端必须是 X 独立式工作台

用户 AI 端承担的是自然语言驱动工作流：

- 新对话
- 录音导入与拜访分析
- 公司分析
- 准备拜访材料
- 我的任务

因此页面必须像正式 AI 工作台，而不是“后台布局里塞一个聊天框”。

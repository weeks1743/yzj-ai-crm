# 0.7.3 金蝶苏州销售习惯会话测试扩展

## 版本目标

- 补充金蝶苏州销售员常用工作习惯下的多轮会话回归场景。
- 覆盖 1 到 10 轮对话，每个轮数 3 条确定性测试用例。
- 从元数据和通用字段语义层修复关系字段表达误写为普通枚举字段的问题。

## 范围说明

- 后端 Agent 记录写入字段语义识别。
- `basicDataWidget` 关系字段的自然语言别名、候选选择和空值澄清。
- `apps/admin-api` 确定性会话 harness 的销售习惯测试语料。

## 关键能力

- “客户信息 / 关联客户 / 绑定客户 / 客户名称 / 客户编号”等表达会按关系字段处理。
- “关联安徽的客户”在联系人上下文中会进入关联客户选择，而不是查询客户或写入省份。
- “更新关联客户”这类有字段意图但缺少目标值的表达会返回远程候选选择卡，不触发空 payload 守卫。
- “查询安徽的客户”和“省改为安徽”仍分别保持为客户查询和省份字段更新。

## 依赖框架

- 主 Agent 内核继续保持业务无关。
- CRM 语义扩展位于 `crm-agent-pack`，依赖记录工具元数据和字段契约。
- 未新增 `scene.*` 运行时技能。

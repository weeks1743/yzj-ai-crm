# 0.8.12 客户简称查询与状态问法修复

## 目标

- 修复 `查询江苏友升`、`查看江苏友升` 进入澄清卡的问题。
- 修复 `江苏友升客户情况`、`江苏友升客户状态`、`江苏友升客户处于什么状态` 把状态/情况词带入客户名称过滤导致查不到的问题。
- 保持 `外部信息` 与 `系统内记录` 的业务无关分层，不修改 Agent core。

## 范围

- 在 CRM 业务包内增加只读客户查询兜底路由：读请求中能抽取出客户简称时，默认选择 `record.customer.search`。
- 对客户状态/客户情况问法清洗主体名，只把客户简称或全称作为客户名称过滤值。
- 不改变 `/公司研究`、外部信息检索、商机/联系人/跟进记录路由。

## 验收

- `查询江苏友升` 走 `record.customer.search`，不再提示缺少目标对象。
- `查看江苏友升` 走 `record.customer.search`，不再提示缺少目标对象。
- `江苏友升客户情况` 过滤值为 `江苏友升`，不是 `江苏友升客户情况`。
- `江苏友升客户处于什么状态` 过滤值为 `江苏友升`，不是整句问题。
- 补充 20 个真实客户简称样本，覆盖 `查询简称` 与 `简称客户状态` 两类用户常见输入：
  `品宏`、`晶镁光罩`、`翔运国际`、`海力风电`、`伯恩医疗`、`星火时空`、`黑崎机械`、`爱朋医疗`、`扬州锻压`、`太訸控股`、`中科离子`、`优格生物`、`比迩泽`、`奥特佳`、`火星河谷`、`友升`、`实极机器人`、`鑫黄产业`、`星瑞包装`、`布朗克斯`。

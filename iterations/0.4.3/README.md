# 0.4.3 版本说明

## 版本目标

- 将后台 `场景组装准备` 重构为 `场景技能` 设计思想页
- 将用户端 `AI 销售工作台` 重排为“双入口工作台”
- 新增复合场景 `拜访后闭环`
- 用 slash 命令静态原型表达统一场景入口

## 本轮范围

- `apps/admin-pro`
- `apps/admin-api`
- `apps/assistant-web`
- `apps/skill-runtime`
- `3rdSkill`
- `packages/shared`
- `docs/04-场景技能总览与编排原则.md`
- `docs/05-录音导入场景设计.md`
- `docs/09-用户对话层与Agent编排.md`
- `iterations/0.4.3/README.md`

## 关键页面 / 能力

### 1. 后台：场景技能

- 菜单文案从 `场景组装准备` 改为 `场景技能`
- 首页改为展示：
  - 什么是场景技能
  - 四个核心对象如何供能
  - 拜访后闭环总流程
  - 按销售阶段组织的复合场景 + 分析场景地图
  - 销售阶段地图改为箭头式流程链路展示，强化阶段推进关系
  - 将 `拜访后第一步`、`需求澄清`、`方案澄清` 在地图层合并为高阶阶段 `拜访后分析`
- 详情页改为设计说明页，突出：
  - 触发入口
  - 适用阶段
  - 组装的记录系统技能
  - 主要外部技能供给
  - 子场景编排链
  - 写回边界
  - 当前场景在销售链路中的位置

### 2. 用户端：AI 销售工作台

- 左侧结构调整为：
  - `AI 销售工作台`
  - `拜访后闭环`
  - `销售主链路`
  - `我的任务`
- 新增独立场景：
  - `拜访后闭环`
- 保留 5 个可独立访问的分析场景：
  - `客户分析`
  - `拜访会话理解`
  - `客户需求工作待办分析`
  - `问题陈述`
  - `客户价值定位`
- 新增方案推进示意场景：
  - `方案匹配与专家协同`
- 输入区和欢迎区增加 slash 命令静态原型：
  - `/拜访后闭环 星海精工股份拜访.mp3`
  - `/客户分析 星海精工股份`
  - `/拜访会话理解 星海精工股份`

## 验收结果

- 后台不再以“依赖缺口治理”作为场景页主叙事
- 用户端首页不再以“公司分析 / 录音导入 / 准备拜访材料”作为一级主入口
- `拜访后闭环` 已成为复合场景入口
- `客户价值定位` 已并入 `拜访后分析` 阶段，不再落在 `方案推进`
- `方案推进` 已补充 `方案匹配与专家协同` 示意场景，用于表达方案、案例和专家资源的组织方式
- slash 命令已成为工作台静态原型中的统一入口语法
- 外部技能目录已切换到销售分析技能集，补齐：
  - `visit-conversation-understanding`
  - `customer-needs-todo-analysis`
  - `customer-value-positioning`
- 已从 `3rdSkill` 和运行时目录中移除：
  - `saas-revenue-growth-metrics`
  - `jobs-to-be-done`
  - `customer-journey-map`
  - `docx`
  - `pdf`
  - `pptx`
  - `xlsx`

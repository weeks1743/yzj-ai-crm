# 0.8.11 外部信息简称检索与数据库兜底

## 目标

- 修复新会话中用公司简称追问外部信息时提示“没有检索到可引用外部信息”的问题。
- 让 `外部信息` 以 MongoDB 中已保存的 Artifact 为事实来源，Qdrant 作为向量检索加速层。
- 保持 Agent core 业务无关，不新增 CRM 场景分支。

## 范围

- `artifact.search` 仍优先使用向量检索和精确 anchor 过滤。
- 当向量无结果、embedding 未配置或 embedding 失败时，回退到同 `eid + appId` 下的 Artifact 元数据匹配。
- 元数据匹配范围包括 `anchors.id`、`anchors.name`、`title` 和 `anchorIdentity`。
- 数据库兜底 evidence 从当前版本 markdown 生成摘要片段，不重新触发公司研究。

## 验收

- 已存 `江苏友升汽车科技有限公司 公司研究` 时，新会话问 `介绍江苏友升的业务` 可命中外部信息。
- `江苏友升有什么业务` 可使用简称匹配，不返回“暂无可用外部信息”。
- 多个 Artifact 同时匹配简称时返回多条 evidence，不静默编造成唯一对象。
- `只看系统内记录` 仍不触发外部信息检索。

## 未完成项

- 本轮不新增官网、新闻、行业报告等外部信息采集源。
- 本轮不修改 `/公司研究` 生成链路。

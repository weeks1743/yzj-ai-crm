# 0.8.13 记录结果隐藏内部 ID

## 目标

- 修复客户查询结果中把 `formInstId` 展示为客户名称的问题。
- 记录卡、读取完成文案和打开详情用户消息都不展示内部 ID。
- 保留 `formInstId` 作为隐藏 `clientAction` 参数，用于后续读取详情。

## 范围

- 后端记录结果标题优先读取字段、`important`、`rawRecord.important` 和顶层展示字段。
- 当真实记录缺少展示字段时，单条结果用用户查询中的对象名称兜底，不回退到内部 ID。
- 前端记录卡和打开详情动作过滤内部 ID；异常旧数据只显示对象名或通用“客户记录/客户详情”。

## 验收

- `查询江苏友升` 命中单个客户时，回复不出现 `69faf11ad03e580001fd4508` 这类内部 ID。
- 客户卡标题展示客户名称或用户输入简称，不展示 `formInstId`。
- 点击“查看详情”发送的可见用户消息为 `打开客户：客户名称` 或 `打开客户详情`，不包含内部 ID。
- `formInstId` 仍保留在隐藏 `record.open` action 中，详情读取能力不受影响。

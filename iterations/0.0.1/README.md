# 0.0.1 版本说明

## 版本目标

搭建 `AI销售助手` 的首版高保真可点击原型，严格遵循 `docs/` 中的产品边界与系统分层。

## 为什么首页是 AI 工作台

根据 [docs/README.md](../docs/README.md) 与 `00-重构总览与产品边界` 的结论，`AI销售助手` 是一个以对话为主入口的销售智能助手系统，用户主要通过 AI 对话完成工作，而不是先进入传统 CRM 列表页。

因此 `0.0.1` 的默认首页必须是：

- `AI工作台 / 对话入口`

而不是：

- 客户列表
- 商机列表
- 传统 dashboard

## 为什么记录系统不是主入口

根据 `docs/README.md` 与 `02-影子系统与核心对象设计`：

- `AI销售助手_记录系统` 是影子系统
- 它承担结构化主数据承载、查询与 AI 确认后写回
- 它不是用户主交互入口

因此 `记录系统` 在原型中定位为次级导航域，用于：

- 查询客户、联系人、商机、跟进记录
- 校对 AI 写回结果
- 查看轻云模板与字段元数据映射

# 根据ticket解析用户身份（单点登录）

**描述：** 此接口都需要通过<a href="docs.html#/server-api/auth/oauth" target="_blank">获取accessToken</a>进行授权, 授权级别为`app`级。 根据客户端生成的`ticket`、企业自建应用`appId`获取当前用户上下文信息。

**请求地址：**  `https://www.yunzhijia.com/gateway/ticket/user/acquirecontext?accessToken=xxxxxx`

**请求方法：** `POST`

**参数类型：** `Content-Type: application/json`

**输入参数：**

字段   | 类型   | 是否必填 | 说明
-------|--------|----------|------------------------------------------------------------------------
appid  | String | 是       | 轻应用id
ticket | String | 是       | 云之家APP会传递ticket参数给轻应用（把ticket参数追加到轻应用对应的url中）,<br>**ticket不可缓存重复使用**
disposable|boolean|是|固定值true


json示例：

```json
{
    "appid":"xxxxxx",
    "ticket":"xxxxxx",
    "disposable":true
}

```

**输出结果：**

```json
{
    "errorCode": 0,
    "data": {
        "appid": "500041933",
        "eid": "10603457",
        "username": "周小迪",
        "userid": "5a41af0ae4b0820e92cd9868",
        "jobNo":"12312",
        "networkid": "58df7e16e4b053f44063edca",
        "deviceId": "xxxx-xxxx-xxxx-xxxx",
        "openid": "5a41b292e4b058cf3d0cf314",
        "ticket": null
    },
    "success": true,
    "error": null
}
```

**参数说明：**

字段      | 类型   | 说明
----------|--------|------------------------------------------------------------------------
appid     | String | 轻应用id
eid       | String | 团队id
username  | String | 用户姓名
userid    | String | 云之家用户id（用户级）
jobNo     | String | 工号
networkid | String | 工作圈id
deviceId  | String | 设备id
openid    | String | 团队用户id(工作圈级，不同的工作圈，该id不同)
ticket    | String | 云之家APP会传递ticket参数给轻应用（把ticket参数追加到轻应用对应的url中）

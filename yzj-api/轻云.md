# 轻云开发API接口


## 1.获取验权令牌

> 本页的接口使用 `team` 级别的 `AccessToken`。

生成令牌所需的appId、secret需要通过具备轻云业务管理员权限的账号通过设计时入口 `管理中心-业务中心-轻云-应用管理-xxx应用-基础设置` 中获取， 然后调用OAuth2获取令牌接口得到team级 `AccessToken`。

![](/opendocs/file/image/feffe7c0005901f7958950ad4bdd2fd2)

如果您还不了解如何获取AccessToken，请<a href="docs.html#/server-api/auth/oauth" target="_blank">点击此处</a>

## 2.如何获取表单codeId

具备轻云业务管理员权限的账号通过设计时入口 `管理中心-业务中心-轻云-应用管理-xxx应用-编辑表单` ， 然后在浏览器url地址栏中可以获取表单formCodeId

![](/opendocs/file/image/1b0f8241dff47f8b9510168f60e55051)

## 3.新增或更新多条无流程单据

**URL：** `https://www.yunzhijia.com/gateway/lightcloud/data/batchSave?accessToken=xxx`

**http请求方法：** `post`

**http内容类型：**`Content-Type:application/json`

**输入参数：**

| 参数 | 类型 | 是否必须 |注释 |
|---|---|---|---|
|eid|String| 是 |工作圈eid |    
|formCodeId|String| 是 |表单codeId |                                             
|oid|String| 是 |新增人openid |
|data|JsonArray| 是 |单据实例数组|

**json示例：**
```json
{
    "eid": "6822727",
    "formCodeId": "e638244f8931481a98897e16674adfbf",
	"oid": "6437a246e4b01c2909637004",
    "data": [
        {
            "formInstId": "6392c76ee3232e50382ea9a6", //单据id，更新单据时必传
            "widgetValue": { //主表单控件字段
                "_S_TITLE": "我是标题111",
                ......
            },
            "details": { 
                "Dd_0": {
                    "widgetValue": [//明细控件字段
                        { // 明细表行内容，key为控件codeId，value为控件值
                            "_id_": "1", // 行id，必填
                            "Te_0": "1",
                            ......
                        }
                    ]
                }
            }
        }
    ]
}
```

**返回结果示例：**

```json
{
    "data":[""],//formInstIds
    "errorCode":0,
    "success":true
}
```

## 6.新增有流程单据

**URL：** `https://www.yunzhijia.com/gateway/lightcloud/data/createProcess?accessToken=xxx`

**http请求方法：** `post`

**http内容类型：**`Content-Type:application/json`

**输入参数：**

| 参数 | 类型 | 是否必须 |注释 |
|---|---|---|---|
|eid|String| 是 |工作圈eid |    
|formCodeId|String| 是 |表单codeId |                                             
|oid|String| 否 |提交人openid |
|oids|String(Array)| 否 |如果是自由流，则填审批人的openid组成的数组，按照顺序进行审批，否则可以不传|
|cc|String(Array)| 否 |自由流抄送人openid数组|
|skipWidgetAuthorityChec|Boolean| 否 |是否跳过节点字段权限校验，true表示本次操作不会校验节点字段权限|
|justDraft|Boolean| 否 |是否保存为草稿，true表示本次提交操作不会创建流程，只是保存为一份草稿(注意：这个存为草稿参数不是暂存到草稿箱，只是给这个发起人暂存了一份数据，在进入发起单据填单界面时自动带出暂存的内容,每个模板每个人最多只会暂存一份数据)|
|useAlias|Boolean| 否 |是否使用别名，true表示使用别名，详情请参考下文对别名的解释|
|resubmit|Boolean| 否 |退回或撤回后是否允许再次发起，true表示允许退回以及撤回到发起节点后再次发起。默认允许退回到发起节点后再次发起|
|widgetValue|JsonArray| 否 |主表单字段赋值|
|details|JsonArray| 否 |明细表字段赋值|

**json示例：**
```json
{
    "eid": "6822727",
    "formCodeId": "043c4167942e4f47b4bba46f0ee01f05",
    "oid":"6437a246e4b01c2909637004", // 有流单据必传
    "oids": ["586db606e4b0913965d37b1a"],
    "cc": ["59cc65d660b2a4b05513da94"], // 选填，自由流抄送人id，key为固定值"cc"，value为由抄送人id(openid)组成的数组
    "skipWidgetAuthorityCheck":true, // 选填，是否跳过节点字段权限校验，true表示本次操作不会校验节点字段权限
    "justDraft":false, // 选填，是否保存为草稿，true表示本次提交操作不会创建流程，只是保存为一份草稿
    "useAlias":false, // 选填，是否使用别名，true表示使用别名，详情请参考下文对别名的解释
    "resubmit":true, // 选填，退回或撤回后是否允许再次发起，true表示允许退回以及撤回后再次发起
    "widgetValue": { // 必填，主表单，key为固定值"widgetValue"
        "_S_TITLE": "fire的自由", // 标题，key固定为"_S_TITLE"，value为字符串格式
        ......
    },
    "details": { // 选填，明细表，key为固定值"details"
        "Dd_0": { // key为明细控件codeId
            "widgetValue": [{ //明细表输入值，key为固定值"widgetValue"
                    "_id_":"1", //行id，必填，值为字符串格式的数字，从1开始累加
                    "Nu_1": "1111.11111",
                    "Mo_2": "22222.22",
                    ......
                }
            ]
        }
    }
}
```

**返回结果示例：**

```json
{
    "errorCode":0,
    "success":true,
    "error":null,
    "data":{
        "formDefId":"5940cc72dce0b4601d2f5e88", // 表单模板id
        "formInstId":"5940cc99a39846705a3a741b", // 单据id
        "flowInstId":"5940cc9dbbd5db466584bd10" // 流程id
    }
}
```

## 7.修改有流程单据

**URL：** `https://www.yunzhijia.com/gateway/lightcloud/data/modifyProcess?accessToken=xxx`

**http请求方法：** `post`

**http内容类型：**`Content-Type:application/json`

**输入参数：**

| 参数 | 类型 | 是否必须 |注释 |
|---|---|---|---|
|eid|String| 是 |工作圈eid |    
|formCodeId|String| 是 |表单codeId |    
|formDefId|String| 是 |表单模板Id | 
|formInstId|String| 是 |单据Id | 
|oid|String| 否 |修改人openid |

```json
{

    "formDefId": "6531e9ed40ef18000192becb",//表单模板唯一id 必传

    "skipWidgetAuthorityCheck":true, // 选填，是否跳过节点字段权限校验，true表示本次操作不会校验节点字段权限
    "useAlias":false, // 选填，是否使用别名，true表示使用别名，详情请参考下文对别名的解释
    "widgetValue": { // 必填，主表单，key为固定值"widgetValue"
        "_S_TITLE": "fire的自由", // 标题，key固定为"_S_TITLE"，value为字符串格式
        ......
    },
    "details": { // 选填，明细表，key为固定值"details"
        "Dd_0": { // key为明细控件codeId
            "widgetValue": [{ //明细表输入值，key为固定值"widgetValue"
                    "_id_":"1", //行id，必填，值为字符串格式的数字，从1开始累加
                    "Nu_1": "1111.11111",
                    "Mo_2": "22222.22",
                    ......
                }
            ]
        }
    },
    "startNode":false
}
```

**返回结果示例：**

```json
{
    "errorCode":0,
    "success":true,
    "error":null,
    "data":{
        "formDefId":"5940cc72dce0b4601d2f5e88", // 表单模板id
        "formInstId":"5940cc99a39846705a3a741b", // 单据id
        "flowInstId":"5940cc9dbbd5db466584bd10" // 流程id

```

## 8.按条件批量查询单据

**URL：** `https://www.yunzhijia.com/gateway/lightcloud/data/searchList?accessToken=xxx`

**http请求方法：** `post`

**http内容类型：**`Content-Type:application/json`

**输入参数：**

| 参数 | 类型 | 是否必须 |注释 |
|---|---|---|---|
|eid|String| 是 |工作圈eid |    
|oid|String| 是 |查询人openid |
|formCodeId|String| 是 |表单codeId |    
|pageNumber|Int| 是 |页码，如果需要查询10000条以上数据请参考：12.10000条以上单据深分页查询遍历|
|pageSize|Int| 是 |分页大小，默认最大100|
|resultItems|JsonArray| 否 |返回字段，目前仅支持控制主表单字段，后续会增加明细字段的控制|
|searchItems|JsonArray| 否 |查询条件|

**json示例：**
```json
{
    "eid": "128152", //必传
    "oid": "61b98ed27be3280001b5dc5b", //必传
    "formCodeId": "3bfff449a0ba4ead97fbf9536096c993", //模板codeId，必传
    "pageNumber": 1,
    "pageSize": 10,
    //可选参数 返回结果需要的字段, 不传默认返回所有字段
    "resultItems": [
       {
             "codeId": "_S_TITLE",
             "parentCodeId": null,
			 "type": "textWidget"
         }
     ],
    //可选参数 条件查询 
    "searchItems": [
        {
            "codeId": "Nu_2",
            "value": ["128142","128142"],
            "type":"numberWidget"
        }
    ]
}
```
**searchItems传参：**
由于控件繁多，请开发者参考轻云列表视图的查询接口进行拼装，任意表单视图列表-F12-点击筛选，关注search2Gen接口入参，如下图：（resultItems、searchItems中只需参考codeId、parentCodeId、value、type这4个属性；特别注意如果是basicDataWidget类型控件作为查询条件需要加入operator操作符作为精确查询条件，如果需要模糊搜索只需传value=对应的展示字段内容；明细里的控件是不支持作为条件直接查询的）
![](/opendocs/file/image/921d0e9c0b77864bd4899e20075b7475)
**返回结果示例：**

```json
{
	"data": {
		"pageNumber": 1,
		"totalPages": 0,
		"pageSize": 10,
		"content": [],
		"totalElements": 0
	},
	"error": null,
	"errorCode": 0,
	"success": true
}
```

## 12.10000条以上单据深分页查询遍历

**URL：** `https://www.yunzhijia.com/gateway/lightcloud/data/scanList?accessToken=xxx`

**http请求方法：** `post`

**http内容类型：**`Content-Type:application/json`

**输入参数：**

| 参数 | 类型 | 是否必须 |注释 |
|---|---|---|---|
|eid|String| 是 |工作圈eid |    
|oid|String| 是 |查询人openid |
|formCodeId|String| 是 |表单codeId |    
|pageSize|Int| 是 |分页大小，默认最大100|
|resultItems|JsonArray| 否 |返回字段，目前仅支持控制主表单字段，后续会增加明细字段的控制|
|searchItems|JsonArray| 否 |查询条件|
|searchAfterId|String| 是 |order=asc时，第一页传0，后续传列表中最后一条的formInstId；order=desc时，第一页传99999999999999999999999999999999，后续同样传列表中最后一条的formInstId|
|order|String| 是 |asc-升序遍历，desc-降序遍历|
![](/opendocs/file/image/a786d7cc38d8a6ad508a0ae0d61271d8)

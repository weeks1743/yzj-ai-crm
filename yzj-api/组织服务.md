# 人员API列表



### 接口授权

本页接口使用 `resGroupSecret` 级别的 `AccessToken`， 与获取组织数据的接口不同，请注意区分。

请在 `管理中心-系统设置-系统集成-通讯录同步` 中，复制 `只读秘钥` 或者 `可编辑秘钥`， 然后调用系统接口换取 `AccessToken`。

> 只读秘钥：只能调用查询类的接口

> 可编辑秘钥：可以调用全部接口

如果您还不了解如何获取AccessToken，请<a href="docs.html#/server-api/auth/oauth" target="_blank">点击此处</a>

## 新增人员

**描述:** 每次新增数不能超过1000条。

**注意:**

- 如果account中的账号传了密码，且是首次导入，则该用户会自动激活，如果没传，则需要用户自行在手机端或网站上激活。
- 作为帐号的email字段通过account字段来传，不作为帐号的email字段通过contact字段来传。
- 如果人员组织不存在，则新建该组织。
- 云之家支持多职位：人员在主职部门的职位，通过更新人员信息接口（updateInfo）去传和更新jobTitle字段（如在主职部门有多个职位，用英文,隔开）；兼职部门的职位，通过设置兼职接口（addPartTimeJobs或addPartTimeJobsFull）去传和更新jobTitle字段（如有多个，用英文,隔开）

**URL:** `https://www.yunzhijia.com/gateway/openimport/open/person/addNew?accessToken=xxxxxx`

**请求方法：** `post`

**内容类型：** `Content-Type: application/x-www-form-urlencoded`

**输入参数:** 

`nonce`: String // 非必填，用于校验重复请求的随机字符串

`eid`: Number // 必填，企业id

`data`: String // 必填，业务数据转换为<font color='red'>json字符串</font>格式


> 请注意，`data` 不应是js对象而应是一个JSON格式的字符串， 即用JSON.stringify()处理以下结构对象获得的字符串：

```json
{
    "eid":String, //不必须，如果没有，则以外面的eid参数为准
    "persons": [
        { //必填,人员列表
            "name":String, //必填,姓名
            "account": {
                "mobile":String, //可选，手机账号。如果account中的email为空，且下面phone字段也为空，那mobile字段不能为空，否则创建账号不成功。如果有mobile字段，也有下面phone字段，则优先以mobile字段作为账号,phone忽略。建议下面phone和mobile只要其中一个传值即可
                "email":String, //可选，邮箱账号
                "password":String //可选，账号密码，如果传了该密码，且是首次导入，则该用户会自动激活，如果没传，则是未激活的账号，需要该用户自行激活
                },
            "phone":String, //可选,手机号码,工作圈内唯一 。如果account字段中mobile和email都为空，那这个phone就必须有，否则创建账号不成功，如果account中的mobile字段存在，则优先以mobile作为账号，phone字段忽略。建议phone和mobile只要其中一个传值即可
            "isHidePhone":String, //可选,是否在通讯录中隐藏手机号码,0: 不隐藏; 1: 隐藏,默认为0
            "department":String, //可选, 部门长名称，格式："一级部门\\二级部门\\三级部门"，如 ： "研发中心\\移动平台产品部\\开发部"，挂在根部门下面，直接传： "\\"，不传该字段则在无部门中
            "jobTitle":String, //可选,主职部门的职位,默认为空,如需同步主职部门兼职职位请使用英文逗号隔开,例："主职职位1,兼职职位1,兼职职位2"，其中第一个逗号前面的职位（主职职位1）为该人的主职职位.
            "jobNo":String, //可选,企业工号
            "gender":String, //可选,性别,0: 不确定; 1: 男; 2: 女,默认为0
            "birthday":String, //可选,生日
            "status":String, //可选,状态 0: 注销，1: 正常，2: 禁用,默认为1
            "weights":int, //可选，排序权重，权重越小，排序越前.
            "orgUserType":int, //可选,是否部门负责人,0表示普通用户，1表示部门负责人
            "contact":jsonarray, //自定义的联系方式，可以存储公司自定义的一些个人信息，比如短号、工号等。contact是公有信息，包括电话，邮箱以及其他三类，每类必须有name,type,value三个属性，type为P支持在手机端直接呼叫打电话,具体格式: [{“name”:”工号”，“type”:”O”, “value”:”GH001”},{“name”:”电话”，“type”:”P”, “value”:”13800000000”},{"name": "邮箱1","type": "E","value": "123aaa@abc.com"}...]其中name是自定义属性的名称，type是类型，type值只有P,E,O三种类型，分别表示手机号，邮箱和其他，value是对应的值
            "hireDate":String,//入职日期，格式如："2018-01-01"
            "positiveDate":String,//转正日期，格式如："2018-01-01"
            "regSource":String //用户来源，如K3OA则对应"K3OA",EAS对应"EAS",K/3对应"K/3",KIS对应"KIS",微信对应"wechat"，S-HR对应"sHR"
            "identityId":String //身份证号，必须为正确的身份证格式
            "staffType":String //人员类型,IN_STAFF：正式员工,OFF_STAFF：非正式员工
        },…
    ]
}
```

## 更新人员信息

**描述:** 根据openId更新人员信息，openId字段必填，其他字段至少有一项不能为空，每次更新数不能超过1000条

**注意:**

- 不再支持人员的新增，新增人员请使用[新增人员接口](#新增人员);
- 更换手机号码请使用[修改手机号接口](#修改手机号);
- 不再支持组织的新增、更改，组织的变更须使用[更新人员组织接口](#更新人员组织);
- 不再支持状态的更改，状态的变更请使用[更新人员状态接口](#更新人员状态);
- 需要更新的字段请将对应的key-value对传过来，不用更新的字段，请不要传递对应的key。举个例子，传"jobTitle":"" 将导致职称被清空（而不是保持职称不修改）。
-  云之家支持多职位：人员在主职部门的职位，通过更新人员信息接口（updateInfo）去传和更新jobTitle字段（如在主职部门有多个职位，用英文,隔开）；兼职部门的职位，通过设置兼职接口（addPartTimeJobs或addPartTimeJobsFull）去传和更新jobTitle字段（如有多个，用英文,隔开）

**URL:** `https://www.yunzhijia.com/gateway/openimport/open/person/updateInfo?accessToken=xxxxxx`

**输入参数:** 

`nonce`: String // 非必填，用于校验重复请求的随机字符串

`eid`: Number // 必填，企业id

`data`: String // 必填，业务数据转换为<font color='red'>json字符串</font>格式


> 请注意，`data` 不应是js对象而应是一个JSON格式的字符串， 即用JSON.stringify()处理以下结构对象获得的字符串


```JSON
{
    "eid":String, //不必须，如果没有，则以外面的eid参数为准
    "persons": [
        { //必填,人员列表
        "openId":String, //必填,人员的openId
        "name":String, //可选,姓名
        "isHidePhone":String, //可选,是否在通讯录中隐藏手机号码,0: 不隐藏; 1: 隐藏,默认为0
        "jobNo":String, //可选,企业工号
        "jobTitle":String, //可选,主职部门的职位,默认为空,如需同步主职部门兼职职位请使用英文逗号隔开,例："主职职位1,兼职职位1,兼职职位2"，其中第一个逗号前面的职位（主职职位1）为该人的主职职位.
        "gender":int, //可选,性别,0: 不确定; 1: 男; 2: 女，默认为 0
        "birthday":String, //可选,生日 清空生日传null 如："birthday":null
        "weights":int, //可选，排序权重，权重越小，排序越前.
        "orgUserType":int, //可选,是否部门负责人,0表示普通用户，1表示设置部门负责人 2 表示取消部门负责人(从部门负责人更新为普通用户时，需传2，传0无效)
        "hireDate":String,//入职日期，格式如："2018-01-01"

            "name": String,

            "value":String
        }...] // 自定义的属性字段集合,更新此字段时即使是部分更新，也需要将全部字段的值也传过来，因为此字段会覆盖保存
        "identityId":String //身份证号，必须为正确的身份证格式
        "staffType":String //人员类型,IN_STAFF：正式员工,OFF_STAFF：非正式员工
```

`nonce`: String // 非必填，用于校验重复请求的随机字符串

`eid`: Number // 必填，企业id

## 更新人员状态

**描述:** 根据openId更新人员状态，每次更新数不能超过1000条

**URL:** `https://www.yunzhijia.com/gateway/openimport/open/person/updateStatus?accessToken=xxxxxx`

**请求方法：** `post`

**内容类型：** `Content-Type: application/x-www-form-urlencoded`

**输入参数:** 

`nonce`: String // 非必填，用于校验重复请求的随机字符串

`eid`: Number // 必填，企业id

`data`: String // 必填，业务数据转换为<font color='red'>json字符串</font>格式


> 请注意，`data` 不应是js对象而应是一个JSON格式的字符串， 即用JSON.stringify()处理以下结构对象获得的字符串


```json
{
    "eid":String, //不必须，如果没有，则以外面的eid参数为准

        "openId":String, //必填,人员的openId
        "type":String, //必填,详见type类型值说明表

```

| type类型值 | 业务说明 | 变更前人员status | 变更后人员status | 接口支持性             |
| ---------- | -------- | ---------------- | ---------------- | ---------------------- |
| 1          | 人员离职 | 1:正常           | 0:注销           | 有效，接口已支持此操作 |
| 2          | 人员恢复 | 0:注销           | 1:正常           | 有效，接口已支持此操作 |
| 3          | 人员禁用 | 1:正常           | 2:禁用           | 有效，接口已支持此操作 |
| 4          | 人员启用 | 2:禁用           | 1:正常           | 有效，接口已支持此操作 |

**URL:** `https://www.yunzhijia.com/gateway/openimport/open/person/updateEmailUnbound？accessToken=xxxxxx`

**请求方法：** `post`

**内容类型：** `Content-Type: application/x-www-form-urlencoded`

**输入参数:** 

`nonce`: String // 非必填，用于校验重复请求的随机字符串

`eid`: Number // 必填，企业id

`data`: String // 必填，业务数据转换为<font color='red'>json字符串</font>格式


> 请注意，`data` 不应是js对象而应是一个JSON格式的字符串， 即用JSON.stringify()处理以下结构对象获得的字符串


```json
{
    "persons": [
        {
            "openId": "577f7e18e4b01f52382636fe",  //用户openid
            "email": "test@123.com"  //邮箱地址
        },
        ...
    ]
}
```
**请求示例：**
```url
https://www.yunzhijia.com/gateway/openimport/open/person/updateEmailUnbound?accessToken=xxxxxx
```

```form
nonce:abcxyXXXXzefg

eid:17951708

data: {
    "persons": [
        {
            "openId": "5de50083e4b09c41c10efe91", 
            "email": "test1@123.com"
        }, 
        {
            "openId": "5df86ebbe4b0b7058a9efc3d", 
            "email": "test2@123.com"
        }
    ]
}

```


**返回结果示例:** 
```json
{
    "success": true,
    "error": "",
    "errorCode": 100,
    "data": [
        {
            "msg": "账号test1@test.com已被5bdfe9e9e4b0f348a44366cd绑定",
            "msgId": "5f72d980e4b022334e7233c2",
            "msgCode": 210,
            "existBind": {
                "openId": "5bdfe9e9e4b0f348a44366cd",
                "email": "test1@123.com"
            }
        }
    ]
}

```
## 获取手机号变更历史记录

**描述:** 获取手机号变更历史记录。

**URL:**`https://www.yunzhijia.com/gateway/openimport/open/person/getPersonChangePhone?accessToken=xxxxxx`

**请求方法：** `post`

**内容类型：** `Content-Type: application/x-www-form-urlencoded`

**输入参数:** 

`nonce`: String // 非必填，用于校验重复请求的随机字符串

`eid`: Number // 必填，企业id

`data`: String // 必填，业务数据转换为<font color='red'>json字符串</font>格式


> 请注意，`data` 不应是js对象而应是一个JSON格式的字符串， 即用JSON.stringify()处理以下结构对象获得的字符串


```json
{
     "begin":int, //可选,计数下标，默认0 即：begin=pageSize(第几页)*pageNum(每页数量)
    "count":int //可选,计数基数, 备：本次获取多少条(建议200~500内),可理解为count=pageSize(每页数量)
    "eid": "2702604", //可选
    "time": "2017-04-02 12:00:00" //必选，查询的起始时间，如："2017-04-02 12:00:00"。（默认的截止时间是当前时间）
}
```

**请求示例：**
```url
https://www.yunzhijia.com/gateway/openimport/open/person/getPersonChangePhone?accessToken=xxxxxx
```

```form
nonce:abcxyXXXXzefg

eid:17951708

data:{
    "begin": 0, 
    "count": 10, 
    "eid": "17951708", 
    "time": "2019-12-17 16:15:00"
}
```

**返回结果示例:** 
```json
{
    "success": true,
    "error": "",
    "errorCode": 100,
    "data": [
        {
            "eid": "17951708",  //工作圈id
            "newPhone": "13013013055",  //更新之后手机号
            "createTime": "2019-12-17 16:18:02",  //时间
            "openId": "5df86ebbe4b0b7058a9efc3d",  //用户openid
            "oldPhone": "13026125321"  //更新之前的手机号
        },
        ...
    ]
}
```

## 根据组织id批量更新人员组织

**描述:** 通过组织id更新人员组织接口(注：当用户为部门负责人时，由于移动部门，原有部门的负责人也随之取消)

**URL:** `https://www.yunzhijia.com/gateway/openimport/open/person/updateDeptByDeptId?accessToken=xxxxxx`

**请求方法：** `post`

**内容类型：** `Content-Type: application/x-www-form-urlencoded`

**输入参数:** 

`nonce`: String // 非必填，用于校验重复请求的随机字符串

`eid`: Number // 必填，企业id

`data`: String // 必填，业务数据转换为<font color='red'>json字符串</font>格式


> 请注意，`data` 不应是js对象而应是一个JSON格式的字符串， 即用JSON.stringify()处理以下结构对象获得的字符串

```json
{
  "eid":String, //不必须，如果没有，则以外面的eid参数为准
  "persons": [
    { //必填,人员列表
        "openId":String, //必填,人员的openId,人员必须在职
        "orgId":String //新的组织id,组织为空则移动到无部门
    },…]
}

```

**请求示例：**
```url
https://www.yunzhijia.com/gateway/openimport/open/person/updateDeptByDeptId?accessToken=xxxxxx
```

```form
nonce:abcxyXXXXzefg

eid:17951708

data:{
    "eid": "17951708", 
    "persons": [
        {
            "openId": "5e0d9cd2e4b02b68c7c5b6ac", 
            "orgId": "11e11baa-d74a-44df-b511-2b0b58d2a19e"
        }, 
        {
            "openId": "5de5003fe4b0b998f379a05e", 
            "orgId": "c37d2483-18c5-4e8b-b89f-1bc7eb862ced"
        }
    ]
}
```

**返回结果示例:** 参见<a href="docs.html#/server-api/org/syncRule.md" target="_blank">输出结果</a>，如果批量更新全部成功，则data里返回[]，如果有未更新成功的，则data中会有未更新成功的具体信息，其中data字段格式如下
```json
[
  {
    "msgId":String, //人员的openId
    "msgCode":int, //消息码
    "msg":String //消息
  },…
]
```

## 添加机密组织可见成员

**描述:** 根据部门Id添加机密组织可见成员，所设置部门需已设置机密组织，否则添加失败

**URL:** `https://www.yunzhijia.com/gateway/openimport/open/person/addOrgSecretPerson?accessToken=xxxxxx`

**请求方法：** `post`

**内容类型：** `Content-Type: application/x-www-form-urlencoded`

**输入参数:** 

`nonce`: String // 非必填，用于校验重复请求的随机字符串

`eid`: Number // 必填，企业id

`data`: String // 必填，业务数据转换为<font color='red'>json字符串</font>格式


> 请注意，`data` 不应是js对象而应是一个JSON格式的字符串， 即用JSON.stringify()处理以下结构对象获得的字符串


```json
{
    "eid":String, //不必须，如果没有，则以外面的eid参数为准
    "orgId":String, //必须
    "secretType":int, //必须，机密组织部门对应类型，1表示隐藏部门，2表示限制该部门人员不可查看其他部门
    "openIds": [String,…] //必填,人员的openId数组（一次最多支持1000个openId）,用“,”号隔开
}
```

**请求示例：**
```url
https://www.yunzhijia.com/gateway/openimport/open/person/addOrgSecretPerson?accessToken=xxxxxx
```

```form
nonce:abcxyXXXXzefg

eid:17951708

data:{
    "eid": "17951708", 
    "orgId": "d66340ca-56ac-443d-b3a3-d9607e81cbba", 
    "secretType": 2, 
    "openIds": [
        "5df8a114e4b02b68c57147e7"
    ]
}
```

**返回结果示例:**

```json
{
    "success": true,
    "error": "",
    "errorCode": 100,
    "data": ""
}

```

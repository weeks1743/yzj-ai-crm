# 智能审批
点击此处查看[视频教程](https://vip.kingdee.com/school/208173411766928640)      
## 简单介绍

智能审批api可实现以下几类场景：  
**1.支持第三方系统通过智能审批api创建流程、修改表单、同意、撤回、退回、废弃、不同意等流程操作**  
**2.支持第三方系统通过智能审批api获取审批单据详情、审批记录、简易流程图、审批状态、模板列表、流程列表等智能审批数据**  
**3.支持通过数据互联控件从第三方获取数据并赋值到智能审批单据**   
**4.支持通过开发者选项将智能审批的数据推送到第三方系统**

### 智能审批sdk(cloudflow-sdk)

cloudflow-sdk提供如下能力：
1. 良好的数据操作能力
2. 便捷的参数校验能力
3. 常用接口的封装能力

**良好的数据操作能力**

表单数据解析步骤较为繁琐。cloudflow-sdk提供了一种良好的表单数据访问能力，该能力提供了一个表单访问器，基于该访问器可以快速访问表单数据(主表单和明细表)。开发者在接收到数据推送后将推送密文交给Cloudflow去进行解密，并得到一个Form对象，该Form对象提供根据控件CodeId、控件Title获取控件值，还可以获取一些基本参数，比如单据实例id、推送事件类型。示例代码如下：
```java
String cipher; // 从开发者选项接收到的推送数据原文
Configuration configuration = new Configuration();
configuration.setCloudflowKey("xxx");
Cloudflow cloudflow = new Cloudflow(configuration);
Form form = cloudflow.decryptAsForm(cipher);
form.getWidgetValueByTitle("姓名");
form.getWidgetValueByCodeId("Ps_0");
form.getDetailWidgetValueByCodeId("Dd_0", "Te_0");
form.getDetailWidgetValueByTitle("明细1", "单行文本");
```

**便捷的参数校验能力**

数据互联控件的返回数据结构较为复杂，只有满足指定规则的返回数据才能被数据互联控件正确解析。cloudflow-sdk提供了一个ApiIntegrationResult对象，基于该对象可以便捷的构建出符合数据互联控件要求的接口返回值。示例代码如下：
```java
	ApiIntegrationResult air = new ApiIntegrationResult();
   
    List<ApiIntegrationResult.Head> heads = new ArrayList<>();
    ApiIntegrationResult.Head head = new ApiIntegrationResult.Head();
    head.setColEnName("name");
    head.setColZhName("姓名");
    head.setShowName(ApiIntegrationResult.ShowName.YES);
    head.setWidgetType(ApiIntegrationResult.WidgetType.TEXT_WIDGET);
    heads.add(head);

    ApiIntegrationResult.Head head2 = new ApiIntegrationResult.Head();
    head2.setColEnName("id");
    heads.add(head2);
    
    air.setColList(heads);

    List<Map<String, Object>> data = new ArrayList<>();
    Map<String, Object> map = new HashMap<>();
    map.put("name", "这是我的名字");
    map.put("id", "1");
    data.add(map);
    air.setDataList(data);

    air.check(); // 校验数据合法性
```

**常用接口的封装能力**

针对常用接口，cloudflow-sdk进行了一系列简单的封装，开发者可以无需知晓内部接口调用细节即可快速调用智能审批开放能力。开发者所需要做的只是提供配置信息并初始化一个Cloudflow对象。示例代码如下：
```java
    Configuration configuration = new Configuration();
	configuration.setCloudflowKey("xxx");
    configuration.setCloudflowAppId("xxx");
    configuration.setCloudflowSecret("xxx");     	
    configuration.setEid("xxx");
    configuration.setHost("https://www.yunzhijia.com");
    cloudflow = new Cloudflow(configuration);
    
    Map<String, String> param = new HashMap<>();
    param.put("formInstId", "xxx");
    param.put("formCodeId", "xxx");
    cloudflow.getFormInstance(param);
```

**别名**

默认情况下审批接口参数都是使用控件的codeId作为参数的key，虽然使用控件的codeId作为参数key能精准的表达参数的含义，但也给开发者带来了一些迁移的麻烦，比如用户在测试工作圈调试时使用的codeId到了正式工作圈可能需要做调整才能正常使用，别名正是为了解决这个麻烦而出现的，用户可以使用别名代替控件的codeId来作为参数的key，这样只要测试工作圈和正式工作圈审批模板里面的控件别名保持一致就可以做到程序从测试环境到正式环境的无缝迁移。
<br />**审批使用控件的标题作为别名。**
<br />**使用别名时要求审批模板内的控件别名不能重复**
<br />**要使用别名只需将参数里面的控件codeId全部替换为别名即可**
<br />**明细表里面的`_id_`字段没有别名**
<br />**明细子控件的别名为明细控件名称 + || + 明细子控件名称**

例如，明细控件名称为`订单明细`，明细子控件为`数量`，则明细子控件别名为`订单明细||数量`

举个例子，使用别名前，调用接口时传参如下
```json
{
    "formCodeId": "be0bac2cbb8f412a91109369b190c8e8",
    "widgetValue": {
        "_S_TITLE": "我是一个小标题",
        "Te_0": "我是个单行文本框"
    },
    "oids": ["586db606e4b0913965d37b1a"],//云之家用户oid
    "creator": "586db606e4b0913965d37b1a"//创建者云之家用户oid
}
```
使用别名后，调用接口时的传参如下
```json
{
    "useAlias":true, // 开启别名
    "formCodeId": "be0bac2cbb8f412a91109369b190c8e8",
    "widgetValue": {
        "标题": "我是一个小标题",
        "单行文本框": "我是个单行文本框"
    },
    "oids": ["586db606e4b0913965d37b1a"],//云之家用户oid
    "creator": "586db606e4b0913965d37b1a"//创建者云之家用户oid
}
```
可以看到参数`widgetValue`中的key已经从控件的codeId变为了控件的标题，但是调用接口得到的效果是一样的。

**保存为草稿**

通过设置`justDraft`参数为true可以将接口参数仅保存为草稿而不会创建流程。对于一个指定的creator和一个指定的formCodeId来说只会有一份有效草稿，即多次调用接口保存为草稿时只有最新那份草稿才是有效的草稿，之前保存的草稿将会被覆盖。通过`justDraft`参数保存为草稿的操作与用户在审批应用内部编辑时产生草稿的方式一致，即通过该接口保存为草稿之后用户从审批页面点击指定模板进去编辑时可以看到草稿数据。   

### 表单修改接口

**关于明细行id的说明**
1. 明细表内容以一个表格形式存在，表格的行即明细表的行，表格的列即明细表的每个字段，行id用于标识明细表行的变化
2. 行id生成规则：行id以字符串形式的数字表示，首行为'1'，第二行为'2'，以此类推，比如初始单据明细表中有2行内容，行id分别为'1'、'2'，编辑过程中在第二行后面添加了一行明细，则新添加的这行明细的行id为'3'。需要注意的是行id只代表明细表行的变化，不能代表行所处的位置，即行id为'3'的行不一定在行id为'2'的后面(在第二行前面插入一行)

    "success":true,

## 获取第三方系统数据

### 数据互联控件说明    

通过数据互联控件，用户可以将其他外部系统的数据赋值给审批表单控件。这需要用户自行开发一个http接口，该接口从外部系统中取数据，然后将数据处理并转换为审批要求的格式并返回给接口调用者。比如将k3 cloud中的数据赋值给表单控件，用户可以开发一个http接口用来从k3 cloud取数据，然后将数据处理并转换为审批要求的格式并返回给接口调用者，需要注意的是该http接口必须能通过外网正常访问。

**FAQ**
```
Q: 为什么赋值规则没生效？
A: 首先检查配置赋值规则之后是否发布了模板，流程是否在发布模板之后发起的；然后检查赋值规则是否有大小写问题，比如赋值规则是将名为KEY的字段赋值给字段Te_0，但实际运行时接口返回的字段名改为了key，这种大小写不匹配的情况会导致赋值失败。
Q: 为什么在搜索框输入关键字后没有搜索效果？
A: 数据互联控件的搜索效果需要由开发者自行实现。即用户在搜索框输入的关键字会通过接口发送给开发者，开发者自行实现搜索逻辑并返回搜索接口。
```

**数据互联控件交互流程**

如果表单处于编辑状态下，当用户点击数据互联控件时审批系统将对用户预先配置的外部数据源接口地址发起http请求，并将接口成功返回的结果赋值给表单相应控件

**使用步骤**

1. 用户自行编写http接口。可以使用任何语言(不限java)编写一个http接口，该接口用来从某个系统中取数据并返回。注意，该http接口必须能通过外网正常访问。
2. 设置表单数据互联控件。在表单模版编辑页面设置一个数据互联控件，并将步骤一编写的http接口地址配置到数据互联控件的外部数据源接口地址中。
3. 校验外部数据源接口地址的返回值格式。用户编写的http接口返回值格式需要符合审批的要求。
4. 设置联动赋值规则。这是一个可选步骤，设置联动赋值规则可以将外部数据源接口地址返回的字段按照对应关系赋值给审批表单控件。
5. 步骤1提供的http接口的contextpath不能为/admin或者/api，比如http://foo.com/admin/bar这种地址会导致请求失败

## 开发者选项数据推送

### 接收数据推送  

**场景说明**
> 外部系统可与外部系统进行集成，接收审批系统消息推送<br/>
> 如合同审批完成后可同步至合同管理系统，请假审批完成后可同步至企业自己的HR系统。<br/>
> 机器人节点场景可与外部系统集成，并根据节点相应设置来获取接口对应的返回值，以此进行下一步操作（继续流转、退回到开始节点等）

**接入审批进行集成开发，开发者需要完成以下步骤**
1. 以审批管理员身份进入审批后台，创建并发布审批模板，审批流程类型要求是固定流程。
2. 在审批节点上注册外部HTTP接口地址。
3. 开发者实现自己的业务逻辑。

**开发者选项须知**
1. 在指定时机触发时，智能审批将向指定接口发起post请求
2. 接口需要在规定时间内返回响应，超时时间为4秒
3. 接口需要返回`success`字符串。*请注意success不要携带任何引号，比如'success'或者"success"都是错误的*
4. 如果接口没有在规定时间内返回规定的响应内容，则系统会判定会推送失败，从而触发重试。重试时间间隔递增，重试最多持续2天
5. 需要确保配置的接口能通过外网访问
6. 接口contextpath不能为/admin或者/api，比如http://foo.com/admin/bar这种地址会导致推送失败
7. 机器人节点的接口返回值格式只有一种（json），与其他节点的开发者选项返回值略有不同

**FAQ**
```
Q：为什么一个事件给我接口推送了多次？
A：这种情况一般是接口没有在规定时间内返回响应，或者是返回了错误的内容，触发了系统重试。按照开发者选项须知的要求处理即可
Q：为什么显示测试数据发送成功，但我却收不到推送？
A：1、确保接口能通过外网访问，
   2、确保没有防火墙之类的设施拦截了请求。
   3、如果接口部署在应用服务器中的话，则检查应用服务器日志。
   4、在接口代码中添加接口日志，记录下接口的请求情况。通过接口日志也能发现问题。
Q：要是我接收服务挂了还能再次接收到推送吗？
A：可以。推送服务有自动重试机制，前3次重试间隔分别为5秒、10秒、30分钟，之后开始每隔一小时重试一次，总共持续2天，直到推送成功或者重试时间结束为止。
Q：为什么从接口日志中看不到推送日志？
A：检查是否有管理员权限，检查流程模板节点上是否设置了开发者选项且已发布。
Q：如何识别重复推送？
A：推送数据中含有eventId，是全局唯一字段，该字段可唯一标识一条推送数据。
```

- 步骤1 创建固定流程

审批管理员进入后台创建新的审批模板， 流程类型选择为固定流程。配置好流程，并发布。

- 步骤2 审批节点或结束节点上配置外部接口

在过程设计器界面，点击节点设置按钮-开发者选项-添加外部接口。开发者可以点击发送测试数据，确认URL是否配置正确。

- 步骤3 开发者实现自己的业务逻辑。

当审批触发指定节点事件时，审批系统将表单数据推送到配置的地址。推送数据经过**加密**处理，然后使用base64编码。开发者需要先使用base64解码，然后进行**解密**处理，解密过程可参考[加密解密说明](#加密解密说明)  （**<font color="red">注意：推送的数据不是标准的json字符串，需通过base64解码，再解密之后，才是标准的json数据！header里面的Content-Type返回的是:application/json</font>**）

> Tips: 审批系统监听相关事件，向配置的接口中发送HTTP请求，失败会有多次补偿重试机制，外部系统需要向审批系统返回成功的标识。标识内容为`success`。<br/>
> 接收方在成功接收数据后先将接收到的数据做持久化处理，然后再异步处理(解密+自定义处理)

### 单据提交前拦截接口
提交前拦截接口是一种特殊的外部接口，使用该接口可以实现对提交(发起人首次提交、审批人审批同意)操作的拦截，同时可以在提交操作真正执行前对表单内容进行修改。

** 场景说明：**

>审批会根据提交前拦截接口的返回值决定该单据能否继续提交。比如有个报销申请单，当报销金额超过10000时无法提交申请，这时用户
可以自行开发一个提交前拦截接口，在接口里面判断报销金额是否超过10000，并根据判断结果返回不同内容，审批将根据该返回值决定
是否继续完成提交操作。

**使用步骤**

1. 在开始节点的开发者选项中添加一个外部接口，其中接口执行时机选择"单据提交前"
2. 发布模版
3. 后续使用该模版发起审批时都会先调用这个外部接口，并根据接口返回结果决定是否继续提交
4. 外部接口的contextpath不能为/admin或者/api，比如http://foo.com/admin/bar这种地址会导致请求失败

**使用说明**

提交前拦截接口的配置与其他外部接口并无差别。当用户使用配置了提交前拦截接口的模版发起审批时，
审批将会对该配置接口发起http调用
>http请求方法：post<br/>
请求头部：Content-Type:application/json;charset=utf-8<br/>
请求地址：用户在外部接口中所填的外部接口地址<br/>
请求参数：使用aes加密后经过base64编码的数据<br/>
要求返回：Content-Type:application/json;charset=utf-8

- 请求参数说明：请求参数与其他外部接口的请求参数一样，都是使用base64编码的经过了aes加密后的数据
用户收到请求参数后应当先使用base64解码，然后使用aes解密即可得到明文参数，解密过程可参考[加密解密说明](#加密解密说明)  （**<font color="red">注意：推送的数据不是标准的json字符串，需通过base64解码，再解密之后，才是标准的json数据！header里面的Content-Type返回的是:application/json</font>**)

- 返回值说明：必须返回如下格式的json字符串
```javascript
{
    "success":true,
    "error":null,
    "data":{
    	"widgetValue":{ // 选填，主表单值，传参格式请参考创建审批接口参数主表单部门
        	// 主表单将按主表单控件采用覆盖形式更新
            // 通过这种方式修改的控件将不受编辑权限约束
        },
        "details":{ // 选填，明细表值，传参格式请参考创建审批接口参数明细表部门
        	// 明细表将按明细控件采用覆盖形式更新
            // 通过这种方式修改的控件将不受编辑权限约束
        }
    }
}
```
字段说明：<br/>

|      字段    |    类型    |  是否必填 |  说明  |
|     ----     |     ----    |      ----    |      ----    | 
|     success      |      boolean    |      必填     |      返回true表示允许继续提交，返回false表示不能继续提交 |
|      error    |      string  |   选填     |      如果success为false，那么error中应当设置一些描述信息以向用户说明为什么不允许提交  |
|      data  |      object  |   选填     |  扩展字段，可以用作修改表单用途  |

如果返回结果中success为true则审批将允许用户完成该次提交操作，否则审批将阻止用户提交，并将error字段中的描述信息展示给用户

**只有http status code为200且返回数据里面的success字段为true才能继续提交。**

**注意，提交前拦截接口必须在3秒钟之内返回响应，否则将被视为请求超时，请求超时同样会阻止用户提交，同时会将超时信息展示给用户。**

**审批请求提交前拦截接口过程中产生的超时、http status code非200等都将视为阻止提交。**

public JSONObject decryptNotification(String cipher) {
    if (cipher == null || cipher.isEmpty()) {
        return null;
    }

    AESEncryptor encryptor = new AESEncryptor(configuration.key);
    String plainText = encryptor.decrypt(cipher);
    return JSON.parseObject(plainText);
}

/**
* php5适用
* @param $enStr 密文
* @param $key 解密密钥
* @return bool|string
*/
function decryptForPhp5($enStr,$key) {
    $decrypted= mcrypt_decrypt(MCRYPT_RIJNDAEL_128,$key,base64_decode($enStr),MCRYPT_MODE_ECB);
    $dec_s = strlen($decrypted);
    $padding = ord($decrypted[$dec_s-1]);
    $decrypted = substr($decrypted, 0, -$padding);
    return $decrypted;
}

## 常见问题 

### 错误码（errorCode）说明

1201001：参数输入错误，请检查参数是否与文档描述的一致  
1201030：当前审批节点为或签，不允许进行加签操作  
1201011：当前登录人非任务处理人（approver已不是当前流程审批人）  
1201029：审批组未开启加签功能（或当前节点不支持加签，如开始节点）  
1201090：当前流程未找到待你审批的任务。说明调用节点错误，当前节点不能执行对应的流程控制

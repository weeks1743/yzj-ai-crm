# 云之家接口映射与待验证清单

## 本篇回答什么问题

本篇回答以下问题：

- `AI销售助手` 设计依赖哪些云之家能力
- 这些能力大致落在哪些官方文档入口
- 哪些接口用途已经明确，哪些还需要联调验证

## 权威文档入口

- 云之家开放平台文档中心：
  <https://open.yunzhijia.com/opendocs/>
- 轻云业务 API：
  <https://open.yunzhijia.com/opendocs/docs.html#/server-api/business/lightcloud>
- 组织 API：
  <https://open.yunzhijia.com/opendocs/docs.html#/server-api/org/index>
- OAuth / Token：
  <https://open.yunzhijia.com/opendocs/docs.html#/server-api/auth/oauth>
- 身份解析：
  <https://open.yunzhijia.com/opendocs/docs.html#/server-api/auth/index>

## 1. 认证与身份解析

### 依赖目的

AI销售助手 需要：

- 获取租户级访问能力
- 解析当前用户身份
- 建立 `eid + appId + userId` 的运行时上下文

### 设计映射

| 能力 | 用途 | 设计依赖 |
|------|------|---------|
| OAuth / Token | 获取调用云之家接口所需 token | 基础设置层、任务系统 |
| 身份解析 | 识别当前登录用户 | 用户上下文、权限与审计 |

### 待验证项

- token 生命周期与刷新机制
- token 是否支持租户 / 应用维度隔离
- 身份解析是否能稳定返回当前用户和租户信息

## 2. 组织同步

### 依赖目的

组织同步用于：

- 拉取部门和人员
- 做 AI销售助手 内部人员映射
- 为销售负责人、跟进负责人、权限控制提供基础

### 设计映射

| 能力 | 用途 | 设计依赖 |
|------|------|---------|
| 组织 API | 获取组织、部门、成员信息 | 组织同步任务、身份映射、权限判断 |

### 待验证项

- 是否支持首次全量 + 后续增量
- 用户离职 / 部门变更如何反映
- 组织同步与轻云应用内用户字段如何对齐

## 3. 轻云业务对象能力

### 依赖目的

轻云业务 API 是影子系统的核心数据来源和回写目标。

它需要支撑：

- 对象元数据读取
- 表单对象实例创建
- 实例查询
- 实例更新
- 关联查询

### 设计映射

| 能力 | 用途 | 设计依赖 |
|------|------|---------|
| 对象 / 模板相关 API | 发现对象与字段结构 | 动态技能中心 |
| 实例查询 API | 查询客户、联系人、商机、跟进记录 | 记录系统技能、场景技能 |
| 实例创建 API | 创建客户、联系人、商机、跟进记录 | 记录系统技能写回 |
| 实例更新 API | 更新对象字段 | 记录系统技能写回 |

### 本项目必须重点验证的对象

- 客户
- 联系人
- 商机
- 商机跟进记录

### 待验证项

- 轻云对象元数据 API 能否完整拿到字段、类型、必填规则、枚举规则
- 是否能稳定识别关联对象字段
- 实例查询的过滤、分页、排序能力是否满足动态技能与问答场景
- 实例写回时是否支持幂等或外部请求号

## 4. 与系统设计的具体关联

### 对基础设置层的关联

- OAuth / Token 配置
- 身份解析配置
- 组织同步配置

### 对影子系统层的关联

- 轻云对象定义读取
- 轻云实例 CRUD

### 对 AI-CRM 层的关联

- 动态技能生成
- 确认后写回
- 对话查询与场景编排

## 5. 接口验证优先级

### P0：必须先验证

- OAuth / Token 获取
- 身份解析
- 轻云对象元数据读取
- 客户 / 联系人 / 商机 / 跟进记录 的实例查询与写回

### P1：建议尽早验证

- 组织同步
- 关联对象字段读取
- 分页、排序、过滤能力

### P2：后续扩展验证

- 更复杂对象
- 报价 / 订单 / 合同链路

## 6. 对文档集的约束

后续文档凡涉及云之家接口时，应遵循以下约束：

1. 写“用途与依赖”，不在设计稿里硬编码未经验证的参数细节
2. 所有对接口的依赖都应能回指到本篇
3. 未验证部分要明确标成待验证，而不是默认为可用

## 本篇结论

云之家接口不是孤立的外部依赖，而是：

- 身份链路的起点
- 影子系统能力的入口
- 动态技能生成的来源
- AI 写回主数据的出口

因此，认证、身份解析、组织同步、轻云业务 API 的联调优先级必须非常靠前，并且需要被文档化为正式的待验证清单。

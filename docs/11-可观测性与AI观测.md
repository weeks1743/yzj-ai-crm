# 可观测性与AI观测

## 本篇回答什么问题

本篇回答以下问题：

- 系统可观测性和 AI 可观测性分别观测什么
- 为什么对 `AI销售助手` 来说，观测不是可选项
- 录音导入、公司分析、准备拜访材料如何做全链路观测

## 总体目标

可观测性的目标不是只监控服务是否活着，而是让团队回答以下问题：

- 这次用户请求走了什么路径
- 智能体选了哪个技能
- 调用了哪些外部系统
- 使用了哪一版研究快照和录音分析结果
- 为什么写回成功或失败
- 哪个租户、哪个用户、哪个任务出的问题

## 推荐组合

当前阶段推荐：

- `OpenTelemetry`
- `Langfuse`
- 应用日志
- 指标监控

### 为什么这样组合

`OpenTelemetry` 负责系统级追踪、span、指标、链路统一。

`Langfuse` 负责 AI 级观测：

- prompt
- model
- tool calls
- latency
- cost
- feedback

## 统一上下文字段

所有日志、追踪、审计、AI 观测必须统一携带：

- `traceId`
- `spanId`
- `eid`
- `appId`
- `userId`
- `threadId`
- `taskId`
- `toolName`
- `modelName`
- `promptVersion`

## 0.7.0 运行洞察与后台观测闭环

`0.7.0` 起，用户 AI 端和管理员后台按同一条智能体运行记录形成闭环：

- 用户 AI 端展示“运行洞察”，面向业务用户解释当前请求的计划、执行状态、工具调用、证据、确认/等待态和策略说明。
- 用户 AI 端只把原始追踪数据放在二级页签，避免把开发排障信息作为默认体验。
- 管理员后台 `Agent 治理 > 运行观测` 读取真实 `agent_runs`、`agent_tool_calls`、`agent_messages` 和 `agent_confirmations`。
- 用户 AI 端通过追踪编号跳转后台运行观测页，管理员可以继续查看完整意图帧、任务计划、执行状态、消息、工具调用和确认审计。

本轮新增只读观测接口：

- `GET /api/agent/runs`
- `GET /api/agent/runs/:runId`
- `GET /api/agent/confirmations`

这些接口只做运行投影，不参与智能体决策，不新增业务对象到主智能体核心层或运行层。

## 需要观测的核心链路

### 1. 基础设置链路

- OAuth 调用
- 身份解析
- 组织同步

### 2. 影子系统链路

- 轻云对象元数据读取
- 轻云对象查询
- 轻云对象写回

### 3. 录音导入链路

- 文件上传
- 哈希去重
- 转写任务
- 实体识别
- 结构化提取
- 候选写回
- 用户确认
- 影子系统回写

### 4. 公司分析链路

- 外部检索
- 原始资料抓取
- 研究摘要生成
- 快照持久化
- 后台刷新

### 5. 准备拜访材料链路

- 数据源读取
- 缺口识别
- 后台补齐
- 简报生成

### 6. 对话层链路

- 智能体决策
- 元工具调用
- 技能调用
- 回答生成

## AI 观测重点

### Prompt 观测

记录：

- 使用了哪版 prompt
- 使用了哪些上下文片段
- 使用了哪些研究快照 / 录音分析版本

### Tool 观测

记录：

- 选了哪个技能
- 入参是什么
- 输出是什么
- 是否成功
- 时延是多少

### 用户反馈观测

记录：

- 用户是否接受写回
- 用户是否采纳建议动作
- 用户是否对结果做纠正

## 场景级观测要求

### 录音导入

必须回答：

- 哪个文件被导入
- 是否命中缓存
- 转写用了哪个引擎
- 绑定了哪个客户 / 商机
- 是否写回跟进记录

### 公司分析

必须回答：

- 研究用了哪些来源
- 用的是历史快照还是新快照
- 研究快照新鲜度如何
- 哪个下游场景消费了这个快照

### 准备拜访材料

必须回答：

- 用了哪些输入来源
- 哪些来源缺失
- 是否触发后台补齐
- 输出引用了哪版研究和哪版录音分析

## 关键指标

### 系统指标

- 请求成功率
- 任务成功率
- 轻云 API 失败率
- 外部研究失败率
- 转写失败率

### AI 指标

- 工具选择成功率
- 澄清触发率
- 写回确认率
- 模型平均时延
- 单任务 token 成本

### 业务指标

- 录音导入完成率
- 公司分析复用率
- 拜访材料生成成功率
- 用户纠错率

## 告警建议

至少配置以下告警：

- 某租户轻云写回连续失败
- 转写任务堆积
- 公司分析来源抓取连续失败
- 某模型调用错误率突增
- 跨租户数据访问被拦截次数异常

## 本篇结论

`AI销售助手` 的可观测性必须同时覆盖：

- 系统运行
- AI 决策
- 业务结果

如果不能回答“这次结果是怎么来的、为什么失败、影响了哪个租户和对象”，那么这套系统就还不具备企业级可运营性。

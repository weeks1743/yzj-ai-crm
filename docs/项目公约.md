# 项目公约

## 1. 版本迭代目录

项目根目录必须维护统一的版本迭代目录：

- 根目录固定使用 `iterations/`
- 每次迭代必须创建对应版本子目录
- 每个版本目录内必须包含说明文件 `README.md`

推荐结构如下：

```text
iterations/
  0.0.1/
    README.md
  0.0.2/
    README.md
```

## 2. 版本说明文件要求

每次迭代都必须编写版本说明文件，用于记录本轮版本的设计目标与实际产出。说明文件至少包含：

- 版本号
- 版本目标
- 范围说明
- 关键页面或能力
- 依赖框架
- 验收结果
- 未完成项与下一步计划

## 3. 版本命名规范

版本号统一采用语义化版本格式：

- 小功能：`0.0.1`
- 中型迭代：`0.1.0`
- 大型里程碑：`1.0.0`

版本升级规则：

- 小功能或局部原型迭代，递增最后一位
- 中型能力组合、跨模块原型收口，递增中间位
- 大范围里程碑或正式阶段切换，递增第一位

## 4. 提交与版本映射

每次代码提交都必须能映射到明确版本，不允许出现“没有版本归属”的提交。

提交要求：

- 每次进入新迭代前先创建对应版本目录与说明文件
- 提交信息必须带版本号，例如：`feat(0.0.1): 初始化 AI 主入口原型骨架`
- 对外发布或阶段收口时应创建对应 tag，例如：`v0.0.1`

## 5. 与本项目设计文档的关系

`docs/` 是本项目的正式设计依据。原型、交互、命名、系统分层都必须优先遵循：

- `0.1.0` 起项目采用“管理员后台 + 用户 AI 端”的双系统协同开发口径
- 两个系统共享同一套租户上下文、记录系统对象、任务资产与对象/技能治理
- `系统设置`、`AI 资产`、`技能与编排` 都必须按正式后台能力组织
- 记录系统能力并入 `技能与编排 > 记录系统对象`，不再作为后台独立一级导航
- 用户 AI 端必须优先采用 `ant-design/x` 独立式工作台范式，而不是把聊天区塞进后台壳体
- `0.5.0` 起主 Agent 内核必须保持业务无关，CRM 只能作为业务工具包、slash 入口、计划模板和验收场景存在
- `0.5.1` 起会话上下文承接必须通过业务无关 `ContextFrame` 与工具元数据实现，不允许为具体 CRM 场景硬编码主 Agent 分支
- 外部技能配置面向普通管理员时，必须优先使用业务语言说明保存、复用、关联和聊天使用方式，不向普通配置页暴露 Artifact、向量、锚点等内部技术概念
- `0.9.0` 起录音能力按“内置录音处理服务产出标准录音资料包，外部技能声明可消费资料”治理；过程 JSON 不进入 Artifact、向量索引、Evidence 或普通对话检索。
- `0.10.3` 起公司分析 Markdown 默认视为可审计资料资产，只承载研究正文、来源、摘要和元数据；下游能力如何复用公司分析 Markdown，必须通过 Agent/工具编排规则和 Skill 输入契约表达，不把“调用策略”写入 Markdown 正文。
- `0.10.3` 起客户拜访准备助手按 `ext.yunzhijia_visit_prep` 外部技能和 `external.yunzhijia_visit_prep` 工具接入，不注册 `scene.visit_prepare` 运行时技能；缺少有效公司研究 Markdown 时必须提示先执行 `/公司研究 公司全称`，不得编造客户研究背景。
- `0.10.4` 起 `/拜访准备` 必须以客户对象为主语，先解析客户，再读取 `客户 -> 公司研究资料`；多个客户或多份公司研究资料命中时使用 Meta Question Card 单选澄清。`ext.yunzhijia_visit_prep` 默认不沉淀资料资产，只返回本轮对话 Markdown；公司研究与录音相关技能继续按静态资料沉淀策略生成资料资产。
- `0.10.4` 起 Agent 运行、资料资产、录音任务、向量检索和对话观测统一以 `eid + config.yzj.lightCloud.appId` 作为隔离标识；`config.yzj.appId` 仅用于 AI 轻应用 SSO/ticket 凭证，不作为资料空间 appId。
- `0.10.7` 起报告生成作为系统内置服务治理：`ext.report_generation` 通过 `report-generation` SKILL 调用仓库内 `apps/report-canvas-service`，输入为公司研究 Markdown，输出为可新页面打开的互动报告 URL；用户端不得用 iframe 嵌入报告。

## 6. 双系统实现约束

为避免再次出现“用了框架名，但形态仍像演示页”的问题，后续版本必须遵守：

- 管理员后台优先使用 `@umijs/max + @ant-design/pro-components`
- 用户 AI 端优先使用 `@ant-design/x + @ant-design/x-sdk + @ant-design/x-markdown`
- 不允许用单文件大拼盘方式手工拼出伪后台或伪聊天原型
- 不允许把旧 `apps/prototype` 继续当作默认主应用增量修补
- 不允许在主 Agent 核心层注册或选择 `scene.*` 运行时技能
